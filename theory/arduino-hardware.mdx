Arduino Nano는 손가락 두 마디 크기의 작은 보드이지만, 그 안에는 프로그램을 실행하는 데 필요한 모든 요소가 갖춰져 있습니다. SAVANT 프로젝트는 이 보드를 5개 사용하여 하나의 폰 노이만 프로세서와 2×2 시스톨릭 어레이를 구현합니다. 이 페이지에서는 아두이노 나노의 하드웨어 구조를 이해하는 데 필요한 핵심 개념을 설명합니다.

## 1. 마이크로컨트롤러란

마이크로컨트롤러(Microcontroller Unit, MCU)는 CPU, 메모리, 입출력 인터페이스를 **하나의 칩에 통합**한 초소형 컴퓨터입니다. 우리가 사용하는 데스크탑 컴퓨터는 CPU, RAM, SSD가 각각 별도의 부품으로 분리되어 있지만, 마이크로컨트롤러는 이 모든 것이 하나의 칩 안에 들어있습니다.

Arduino Nano에 탑재된 **ATmega328P**의 주요 사양은 다음과 같습니다.

| 구성 요소 | 사양 | 역할 |
|-----------|------|------|
| CPU | 8비트 AVR, 16MHz | 명령어 실행 |
| 플래시 메모리 | 32KB | 프로그램 저장 (전원 꺼져도 유지) |
| SRAM | 2KB | 변수 저장 (전원 끄면 초기화) |
| EEPROM | 1KB | 소량의 데이터 영구 저장 |

<Note>
일반 PC의 CPU는 수백 MHz~수 GHz 클럭에 수 GB의 RAM을 사용합니다. ATmega328P는 이보다 훨씬 느리고 메모리도 적지만, 가격이 저렴하고 소비 전력이 낮으며 크기가 작아 임베디드 시스템에 널리 사용됩니다.
</Note>

## 2. Arduino Nano 핀 구조

Arduino Nano의 핀은 크게 디지털 핀, 아날로그 핀, 전원 핀으로 나뉩니다.

### 디지털 핀 (D0 ~ D13)

디지털 핀은 **HIGH(5V) 또는 LOW(0V)** 두 가지 값만 출력하거나 읽을 수 있습니다.

| 핀 번호 | SAVANT에서의 역할 |
|---------|-----------------|
| D0 (RX) | 하드웨어 시리얼 수신 — USB 통신용으로 예약됨 |
| D1 (TX) | 하드웨어 시리얼 송신 — USB 통신용으로 예약됨 |
| D2 ~ D5 | 보드 간 통신 (CLOCK/DATA 신호선) |
| D6 | 타이밍 신호 (PE(0,0) → PE(1,1) 동기화) |
| D7, D8 | LED 상태 표시 (선택적 사용) |
| D3, D5, D6, D9, D10, D11 (~표시) | PWM 출력 가능 |

<Note>
D0(RX)와 D1(TX)는 USB를 통해 컴퓨터와 통신할 때 사용되는 하드웨어 시리얼 핀입니다. SAVANT에서는 이 핀들을 보드 간 통신에 사용하지 않고, D2~D5 핀을 커스텀 클럭 방식 통신에 사용합니다.
</Note>

### 아날로그 입력 핀 (A0 ~ A7)

아날로그 핀은 0V~5V 사이의 **연속적인 전압값**을 0~1023 사이의 숫자로 읽을 수 있습니다.

| 핀 번호 | 특수 기능 | SAVANT에서의 역할 |
|---------|----------|-----------------|
| A4 (SDA) | I2C 데이터 | LCD 모듈 연결 |
| A5 (SCL) | I2C 클럭 | LCD 모듈 연결 |
| A0 ~ A3 | — | 필요 시 추가 입력용 |

## 3. 전원 공급 방법

Arduino Nano에 전원을 공급하는 방법은 세 가지입니다.

| 방법 | 입력 전압 | 설명 |
|------|----------|------|
| USB (Micro 5핀) | 5V | 가장 간단한 방법. PC 또는 USB 허브에서 직접 공급 |
| VIN 핀 | 7 ~ 12V | 외부 전원 어댑터 연결. 보드 내 레귤레이터가 5V로 변환 |
| 5V 핀 | 5V | **직접 외부 전원 연결 금지** |

<Warning>
**5V 핀에 외부 전원을 직접 연결하지 마십시오.** 5V 핀은 레귤레이터를 거치지 않고 보드 내부 회로에 직결됩니다. 전압이 조금이라도 높으면 ATmega328P 칩이 영구적으로 손상될 수 있습니다. 외부 전원을 공급할 때는 반드시 VIN 핀(7~12V) 또는 USB를 사용하십시오.
</Warning>

**SAVANT 전원 구성:** 5개의 아두이노 나노 보드를 모두 USB 허브에 연결합니다. USB 허브가 각 보드에 5V 전원을 공급하며, 동시에 시리얼 모니터 접속도 가능합니다.

## 4. 디지털 신호

보드 간 통신에 사용되는 디지털 핀 조작의 기본 함수입니다.

```cpp
// 핀 모드 설정 (setup()에서 한 번만 실행)
pinMode(2, OUTPUT);  // D2 핀을 출력 모드로
pinMode(3, INPUT);   // D3 핀을 입력 모드로

// 신호 출력
digitalWrite(2, HIGH);  // D2 핀에 5V 출력
digitalWrite(2, LOW);   // D2 핀에 0V 출력

// 신호 읽기
int state = digitalRead(3);  // D3 핀 값 읽기 (HIGH=1, LOW=0)
```

SAVANT의 보드 간 통신은 이 세 가지 함수를 반복적으로 호출하여 1비트씩 데이터를 전송합니다.

## 5. 시간 측정 함수

SAVANT 실험의 핵심은 폰 노이만 방식과 시스톨릭 어레이 방식의 **처리 시간을 정밀하게 측정**하는 것입니다. 아두이노는 두 가지 시간 측정 함수를 제공합니다.

| 함수 | 단위 | 특징 |
|------|------|------|
| `millis()` | 밀리초 (ms) | 전원 켠 이후 경과 시간, 일반적인 지연 측정에 적합 |
| `micros()` | 마이크로초 (μs) | 더 정밀한 측정, SAVANT 실험에서 사용 |

```cpp
// SAVANT에서 사용하는 실행 시간 측정 패턴
unsigned long start = micros();   // 측정 시작

// --- 측정할 연산 실행 ---
performMatrixMultiplication();
// -------------------------

unsigned long elapsed = micros() - start;  // 경과 시간 계산

Serial.print("소요 시간: ");
Serial.print(elapsed);
Serial.println(" 마이크로초");
```

<Tip>
`millis()`와 `micros()`는 모두 `unsigned long` 타입을 반환합니다. 측정 결과를 저장할 변수도 반드시 `unsigned long`으로 선언해야 오버플로우 없이 올바른 값을 얻을 수 있습니다.
</Tip>

## 6. 아두이노 코드 구조

아두이노 스케치(프로그램)는 반드시 두 함수를 포함합니다.

```cpp
void setup() {
  // 전원이 켜지거나 리셋될 때 단 한 번 실행됨
  // 핀 모드 설정, 시리얼 통신 초기화 등
  Serial.begin(9600);
  pinMode(2, OUTPUT);
}

void loop() {
  // setup() 종료 후 무한 반복 실행됨
  // 일반적인 아두이노 프로그램의 주요 로직이 여기에 위치
}
```

**SAVANT에서의 구조:** SAVANT는 단발성 측정을 수행하므로, 행렬 연산과 시간 측정 코드를 모두 `setup()` 안에 작성합니다. `loop()`는 비워두거나 단순 대기 상태로 유지합니다.

```cpp
void setup() {
  Serial.begin(9600);
  // 핀 초기화 ...

  unsigned long start = micros();
  // 행렬 연산 수행 ...
  unsigned long elapsed = micros() - start;

  Serial.println(elapsed);
}

void loop() {
  // 아무것도 하지 않음 — 측정은 setup()에서 완료됨
}
```

<Tip>
측정을 반복하고 싶을 때는 코드를 다시 업로드할 필요 없이 **리셋 버튼**을 누르면 됩니다. 리셋 버튼은 보드를 전원 재투입한 것과 동일하게 `setup()`부터 다시 실행합니다.
</Tip>

## 7. CH340 드라이버

Arduino Nano 호환 보드(SZH-EK025 등)는 USB-시리얼 변환 칩으로 **CH340**을 사용합니다. 이 칩이 있어야 컴퓨터가 아두이노 보드를 USB 포트로 인식할 수 있습니다.

<Note>
정품 Arduino Nano는 FTDI 또는 ATmega16U2 칩을 사용하지만, 국내에서 많이 유통되는 저가형 호환 보드는 CH340 칩을 사용합니다. 두 종류는 기능상 동일하지만 드라이버가 다릅니다.
</Note>

**Windows 설치 순서:**

1. CH340 드라이버 파일(`CH341SER.EXE`)을 제조사 또는 신뢰할 수 있는 경로에서 다운로드합니다.
2. 설치 프로그램을 실행하고 "INSTALL" 버튼을 클릭합니다.
3. 아두이노 보드를 USB로 연결합니다.
4. 장치 관리자(Device Manager)를 열어 **포트(COM & LPT)** 항목에 `USB-SERIAL CH340 (COMx)`가 표시되는지 확인합니다.
5. Arduino IDE에서 **도구 → 포트** 메뉴에서 해당 COM 포트를 선택합니다.

<Warning>
드라이버를 설치하지 않으면 Arduino IDE에서 보드를 인식하지 못해 코드 업로드가 불가능합니다. 보드를 처음 연결할 때 COM 포트가 나타나지 않으면 CH340 드라이버 설치 여부를 먼저 확인하십시오.
</Warning>
