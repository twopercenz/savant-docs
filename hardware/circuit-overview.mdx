SAVANT의 병렬 처리 그룹은 Arduino Nano 4개를 2×2 격자 형태로 배치한 시스톨릭 어레이로 구성됩니다. 각 보드는 처리 요소(Processing Element, PE)로서 독립적으로 연산을 수행하며, 인접 보드와 디지털 핀을 통해 직접 데이터를 주고받습니다. 별도의 단일 보드(Single Board)는 Von Neumann 순차 처리 대조군으로 사용되며, 시스톨릭 격자와는 신호선 연결 없이 독립적으로 동작합니다.

## 격자 배치 구조

4개의 PE 보드는 아래와 같이 2행 2열로 배치됩니다. 오른쪽 끝 열 아래에 16×2 I2C LCD가 연결되어 최종 연산 결과와 경과 시간을 표시합니다.

```
PE(0,0) ─── PE(0,1)
  │              │
PE(1,0) ─── PE(1,1)  ←── LCD 16×2 (I2C)

Single Board (대조군) — 격자와 분리, 신호선 없음
```

각 PE의 좌표는 `(행, 열)` 순서로 표기합니다. PE(0,0)은 격자의 좌상단에 위치하며, 시스톨릭 연산의 시작점이자 타이밍 신호를 발생시키는 주 제어 보드 역할을 겸합니다.

## 데이터 흐름 방향

시스톨릭 어레이에서 행렬 곱셈을 수행하기 위해 두 종류의 데이터가 서로 다른 방향으로 흐릅니다.

**A값 (가로 흐름, 왼쪽 → 오른쪽)**
- PE(0,0) → PE(0,1): 0행의 A 데이터 전달
- PE(1,0) → PE(1,1): 1행의 A 데이터 전달

**B값 (세로 흐름, 위쪽 → 아래쪽)**
- PE(0,0) → PE(1,0): 0열의 B 데이터 전달
- PE(0,1) → PE(1,1): 1열의 B 데이터 전달

각 PE는 수신한 A값과 B값을 곱하여 자신의 누적 합산값(C값)에 더합니다. 이 구조 덕분에 4개의 PE가 동시에 서로 다른 부분 적을 계산하므로, 순차 처리 대비 이론적으로 최대 4배의 처리 속도 향상이 가능합니다.

## D6 타이밍 신호

시스톨릭 연산의 실제 소요 시간을 측정하기 위해 PE(0,0)의 D6 핀에서 PE(1,1)의 D6 핀으로 별도의 타이밍 신호선이 연결됩니다.

<Note>
PE(0,0)은 연산 시작 직전 D6 핀을 `HIGH`로 설정하고, PE(1,1)은 모든 연산이 완료된 시점에 D6 핀의 상태 변화를 감지하여 경과 시간을 측정합니다. 이 방식은 `millis()` 또는 `micros()` 함수를 단일 보드 내에서만 호출하는 것보다 분산 환경에서의 실제 지연을 더 정확하게 포착합니다.
</Note>

## LCD 연결 및 표시 목적

PE(1,1)의 아날로그 핀 A4(SDA)와 A5(SCL)에 16×2 I2C LCD가 연결됩니다. LCD는 시스톨릭 연산이 완료된 후 다음 두 가지 정보를 표시합니다.

- **결과 행렬**: 4개의 C값 (각 PE가 계산한 부분 적의 최종 합)
- **경과 시간**: D6 타이밍 신호를 기준으로 측정한 시간(μs 단위)

LCD를 사용하는 이유는 모든 보드가 유전원 USB 허브에 연결되기 때문입니다. 허브에 연결된 상태에서는 PC의 시리얼 모니터를 5개 보드 모두에 동시 열기가 어려우므로, PE(1,1)의 결과를 LCD에 직접 출력하는 방식을 선택했습니다.

## 연결 구성 요약

| 구분 | 수량 |
|------|------|
| 신호선 (A 흐름 CLOCK/DATA) | 4개 |
| 신호선 (B 흐름 CLOCK/DATA) | 4개 |
| D6 타이밍 신호선 | 1개 |
| **신호선 합계** | **9개** |
| GND 공통 연결 (보드 GND → 공통 레일) | 5개 |
| LCD I2C 연결선 (SDA, SCL, VCC, GND) | 4개 |

## 핵심 설계 원칙

### 커스텀 클록 동기 통신

SAVANT는 표준 라이브러리인 `SoftwareSerial`을 사용하지 않고, Arduino 내장 함수 `shiftOut()`과 커스텀 `receiveByte()` 함수를 이용한 클록 동기 직렬 통신을 구현합니다.

<Note>
`SoftwareSerial`은 보드 간 독립 타이밍에 의존하는 비동기 방식입니다. 여러 보드가 동시에 수신을 시도할 경우 인터럽트 충돌과 타이밍 오차가 발생할 수 있습니다. SAVANT의 클록 동기 방식은 송신 측이 클록 신호를 직접 제어하므로 타이밍 오차 없이 신뢰성 있는 데이터 전달이 가능합니다.
</Note>

각 신호 연결은 CLOCK 핀과 DATA 핀, 2개의 선으로 구성됩니다.
- **송신**: `shiftOut(DATA_PIN, CLOCK_PIN, MSBFIRST, value)`
- **수신**: 커스텀 `receiveByte(clockPin, dataPin)` — 클록 핀의 상승 에지를 폴링하여 비트를 순서대로 읽음
- **PE(1,1)의 이중 수신**: A와 B 두 방향에서 동시에 수신해야 하므로, 두 CLOCK 핀을 동시에 폴링하는 전용 루프를 사용

### 공통 GND 공유

신호 통신의 기준 전압을 일치시키기 위해 5개 보드 모두의 GND 핀을 브레드보드의 공통 GND 레일에 연결합니다. USB 허브의 USB 케이블을 통해서도 GND가 공유되므로 이중 연결이 형성되지만, 이는 오히려 접지 안정성을 높이는 효과가 있습니다.

<Tip>
GND 공유가 누락되면 보드 간 신호 전압 기준이 달라져 데이터 수신 오류가 발생합니다. 배선 완료 후 멀티미터로 모든 보드의 GND 핀 간 저항을 측정하여 공통 연결 여부를 반드시 확인하세요.
</Tip>
