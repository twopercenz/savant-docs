SAVANT는 5개의 Arduino Nano 보드를 동시에 구동해야 하므로, 단일 PC USB 포트만으로는 안정적인 전원 공급이 어렵습니다. 이 문제를 해결하기 위해 **유전원(전원 어댑터 포함) USB 허브**를 사용합니다. 유전원 허브는 각 포트에 독립적인 전류를 공급할 수 있어 5개 보드 모두를 안정적으로 구동하며, USB 케이블을 통한 GND 공유라는 부가 이점도 제공합니다.

## 유전원 USB 허브를 권장하는 이유

일반 PC의 USB 포트는 허브 전체 합산 기준 약 500mA~900mA의 전류를 공급합니다. Arduino Nano 1개는 동작 중 약 20~40mA를 소비하므로 5개를 동시 연결하면 순간 부하가 증가하여 전압 강하나 리셋이 발생할 수 있습니다.

유전원 USB 허브는 별도의 AC 어댑터로부터 전원을 공급받아 **각 USB 포트에 최대 500mA~1A를 독립적으로** 제공합니다. 이를 통해 다음 이점을 얻을 수 있습니다.

| 항목 | PC 직결 USB | 유전원 USB 허브 |
|------|------------|----------------|
| 포트당 전류 용량 | 공유 (불안정) | 독립 (안정) |
| 5개 보드 동시 구동 | 전압 강하 위험 | 안정적 구동 |
| GND 공유 | 단일 포트만 | 모든 포트 공통 GND |
| 코드 업로드 | 가능 | 가능 (동시 인식) |

## USB 허브를 통한 GND 자동 공유

유전원 USB 허브에 연결된 모든 장치는 USB 케이블의 GND 선을 통해 공통 GND를 공유합니다. SAVANT에서는 이에 더해 각 보드의 GND 핀을 브레드보드 공통 GND 레일에 별도로 연결하므로, GND 경로가 두 가지(USB 케이블 경로 + 핀 레일 경로)가 됩니다.

<Note>
이중 GND 연결은 문제가 아닙니다. GND는 전위 기준점이므로 여러 경로로 연결될수록 접지 저항이 낮아지고 신호 안정성이 높아집니다. 특히 클록 동기 통신에서 GND 품질은 데이터 수신 오류율에 직접적인 영향을 미치므로, 이중 연결은 오히려 권장되는 설계입니다.
</Note>

## 코드 업로드 절차

유전원 허브에 연결된 상태에서 Arduino IDE로 코드를 업로드할 수 있습니다. 단, 다음 순서를 따르면 포트 충돌 없이 업로드할 수 있습니다.

<Steps>
  <Step title="각 보드를 노트북에 개별 연결하여 코드 업로드">
    처음 코드를 업로드할 때는 각 보드를 노트북 USB 포트에 **개별적으로** 연결합니다. Arduino IDE에서 올바른 포트(COM 포트 또는 /dev/tty*)와 보드 종류(Arduino Nano)를 선택한 후 업로드하세요. 업로드 완료 후 케이블을 뽑기 전에 시리얼 모니터로 초기 동작을 확인합니다.
  </Step>
  <Step title="업로드 완료 보드를 유전원 허브로 이전">
    코드 업로드가 완료된 보드부터 순서대로 유전원 허브의 USB 포트에 연결합니다. 허브 연결 후에도 Arduino는 업로드된 코드를 그대로 유지하며 자동으로 실행됩니다.
  </Step>
  <Step title="PE(0,0) 최종 업로드 및 시리얼 모니터 확인">
    PE(0,0)은 시스톨릭 연산의 시작 신호를 보내는 보드입니다. 마지막으로 업로드한 후 시리얼 모니터를 열어 입력 행렬 값이 올바르게 출력되는지 확인합니다. 이후 PE(0,0)도 허브로 이전합니다.
  </Step>
  <Step title="모든 보드 허브 연결 완료 후 전체 실행">
    5개 보드가 모두 허브에 연결되면 허브 전원을 켜거나 허브를 PC에 연결합니다. 모든 보드가 동시에 부팅되며, PE(0,0)이 연산 시작 조건을 감지하면 시스톨릭 어레이가 동작을 시작합니다. PE(1,1)에 연결된 LCD에 최종 결과와 경과 시간이 표시됩니다.
  </Step>
</Steps>

## 시리얼 모니터가 필요한 보드

전체 실행 중 시리얼 모니터를 통해 중간 값을 확인하고 싶은 경우, 다음 우선순위를 참고하세요.

| 보드 | 시리얼 모니터 용도 | 필요성 |
|------|--------------------|--------|
| PE(0,0) | 입력 행렬 A, B 값 확인 및 연산 시작 시점 출력 | 높음 (디버깅 시) |
| PE(1,1) | 결과 행렬 C 값 및 경과 시간 | **LCD로 대체 가능** |
| PE(0,1), PE(1,0) | 수신된 A 또는 B 값 중간 확인 | 낮음 (정상 동작 후 불필요) |
| Single | 순차 처리 소요 시간 및 결과 | 높음 (대조군 측정) |

<Note>
PE(1,1)은 LCD를 통해 결과를 직접 표시하므로, 허브에 연결된 상태에서도 시리얼 모니터 없이 결과를 확인할 수 있습니다. 이것이 LCD를 채택한 주된 이유입니다.
</Note>

## 배터리 전원 (대안, 비권장)

USB 허브를 사용할 수 없는 환경에서는 9V 배터리를 Arduino Nano의 **VIN 핀**에 연결하는 방법이 있습니다. Arduino Nano의 내장 레귤레이터가 9V를 5V로 변환하여 보드를 구동합니다.

<Warning>
9V 배터리를 **5V 핀에 직접 연결하면 보드가 즉시 손상됩니다.** 배터리 전원은 반드시 VIN 핀에만 연결하세요. 5V 핀은 레귤레이터를 거치지 않는 직통 전원 핀으로, 허용 전압 범위를 초과하는 전압이 인가되면 마이크로컨트롤러와 보드 부품이 돌이킬 수 없이 파손됩니다.
</Warning>

배터리 전원 방식은 다음 이유로 SAVANT 프로젝트에서 권장하지 않습니다.

- 9V 배터리의 내부 저항으로 인해 부하 시 전압이 떨어져 보드 리셋이 발생할 수 있습니다.
- 5개 보드에 각각 배터리를 연결해야 하므로 비용과 부피가 증가합니다.
- 배터리 방식에서는 GND 공유를 위해 별도의 GND 연결선을 모든 보드 간에 명시적으로 구성해야 합니다.
- 코드 업로드를 위해 어차피 USB 연결이 필요하므로 허브 방식과 비교해 실질적인 이점이 없습니다.

<Tip>
현장 시연이나 이동 중 테스트가 필요하다면 배터리 대신 고출력 보조 배터리(Power Bank)와 USB 멀티 포트 어댑터를 조합하는 것이 더 안전하고 안정적입니다.
</Tip>
