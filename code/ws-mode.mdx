WS(Weight Stationary) 모드는 각 PE(Processing Element)가 자신이 담당하는 가중치 행렬 원소를 펌웨어에 하드코딩해 고정하고, 오직 입력값(B 행렬 원소)만 격자 사이를 흘려보내는 방식입니다. OS 모드에서는 A·B·부분합 세 종류의 값이 이동하지만, WS 모드에서는 가중치를 전송할 필요가 없으므로 통신 횟수가 줄어들고, 각 PE의 역할이 단순해집니다. 아래에서 WS 모드의 동작 원리, 보드별 역할, 전체 코드, 그리고 OS 모드에서 전환하는 방법을 순서대로 설명합니다.

## WS 모드 vs OS 모드: 핵심 차이

| 항목 | OS 모드 | WS 모드 |
|------|---------|---------|
| 가중치(A) 전송 | PE 간 전달 | 각 PE에 고정(하드코딩) |
| 입력(B) 전송 | PE 간 전달 | PE 간 전달 |
| 부분합 전송 | PE 간 전달 | PE 간 전달 |
| 통신 데이터 종류 | A, B, 부분합 | B, 부분합 |
| 모드 전환 방법 | 다른 코드 세트 업로드 | 다른 코드 세트 업로드 |

<Note>
WS 모드에서는 PE(0,0)이 B 열의 값만 오른쪽·아래로 전송합니다. A 행렬의 원소는 전송되지 않고, 각 PE의 펌웨어에 상수로 고정됩니다.
</Note>

## 보드별 역할

기본값: A = [[2, 3], [4, 5]], B = [[1, 2], [3, 4]], 기댓값 C = [[11, 16], [19, 28]]

WS 모드에서 C[1][j] = A[1][0]×B[0][j] + A[1][1]×B[1][j]입니다. 이를 올바르게 계산하려면 PE(1,0)이 B의 **첫 번째 행** 값(B[0][j])을 받아야 하고, PE(1,1)은 PE(1,0)의 부분합과 B의 **두 번째 행** 값(B[1][j])만을 누산합니다. PE(0,1)의 부분합은 C[0][j]에 속하므로 PE(1,1)으로 전달되지 않습니다.

| 보드 | 고정 가중치 | 받는 값 | 보내는 값 |
|------|-----------|---------|----------|
| PE(0,0) | W[0][0] = 2 | 없음 (마스터) | B[0][p] → PE(0,1) (오른쪽), B[0][p]+B[1][p] → PE(1,0) (아래) |
| PE(0,1) | W[0][1] = 3 | B[0][p] ← PE(0,0) | 없음 (C[0][p] 자체 계산 종단) |
| PE(1,0) | W[1][0] = 4 | B[0][p]+B[1][p] ← PE(0,0) | 부분합(4×B[0][p]) + B[1][p] → PE(1,1) (오른쪽) |
| PE(1,1) | W[1][1] = 5 | 부분합+B[1][p] ← PE(1,0) | 최종 C[1][p] LCD 출력 |

<Note>
C[0][p] 결과는 PE(0,1)이 자체적으로 계산합니다. 해당 값은 LCD에는 표시되지 않으며, PE(0,1)의 시리얼 포트를 순간적으로 노트북에 연결하면 시리얼 모니터로 확인할 수 있습니다.
</Note>

## 핀 배치 (전 보드 공통)

| 핀 | 역할 |
|----|------|
| D2 (RX_SW) | SoftwareSerial 수신 |
| D3 (TX_SW) | SoftwareSerial 송신 |
| D4 | 제2 SoftwareSerial 수신 (PE(0,0), PE(1,0) 전용) |
| D5 | 제2 SoftwareSerial 송신 (PE(0,0), PE(1,0) 전용) |
| D6 | 타이밍 신호 (PE(0,0) 출력 / PE(1,1) 입력) |
| A4 (SDA) | I2C LCD — PE(1,1) 전용 |
| A5 (SCL) | I2C LCD — PE(1,1) 전용 |

## 전체 코드

### PE(0,0) — 마스터, D6 타이밍 신호 출력

PE(0,0)은 B 행렬의 열 값을 하나씩 오른쪽(PE(0,1))과 아래(PE(1,0))로 전송합니다. 자신의 MAC 연산 직전에 D6를 HIGH로 설정하여 타이밍 측정을 시작합니다.

```cpp
// WS 모드 — PE(0,0)
// 고정 가중치: W[0][0] = 2
// B[0][p]를 PE(0,1)(오른쪽)으로 전송
// B[0][p]와 B[1][p] 모두를 PE(1,0)(아래)으로 전송
// C[1][p] = A[1][0]*B[0][p] + A[1][1]*B[1][p] 계산에 필요한 두 값을 PE(1,0)이 모두 받아 처리

#include <SoftwareSerial.h>

// 오른쪽(PE(0,1))으로 전송
SoftwareSerial serRight(2, 3); // RX=D2, TX=D3

// 아래(PE(1,0))로 전송
SoftwareSerial serDown(4, 5);  // RX=D4, TX=D5

#define PIN_D6 6   // 타이밍 신호 출력

// 고정 가중치
const int W = 2;  // W[0][0] = A[0][0]

// B 행렬 (열 단위로 저장)
// B = [[1,2],[3,4]]
const int B_row0[] = {1, 2};   // B[0][0], B[0][1]
const int B_row1[] = {3, 4};   // B[1][0], B[1][1]

#define P_SIZE 2   // B의 열 수

void setup() {
  Serial.begin(9600);
  serRight.begin(9600);
  serDown.begin(9600);

  pinMode(PIN_D6, OUTPUT);
  digitalWrite(PIN_D6, LOW);

  delay(500); // 다른 보드 초기화 대기
  Serial.println("[PE(0,0)] WS mode ready.");
}

void loop() {
  for (int p = 0; p < P_SIZE; p++) {
    int b0 = B_row0[p];  // B[0][p]: 첫 번째 행
    int b1 = B_row1[p];  // B[1][p]: 두 번째 행

    // 자신의 MAC 직전에 D6 HIGH (타이밍 측정 시작)
    if (p == 0) digitalWrite(PIN_D6, HIGH);

    // PE(0,0) 자신의 MAC: W[0][0] × B[0][p]
    int mac_00 = W * b0;
    Serial.print("[PE(0,0)] p="); Serial.print(p);
    Serial.print(" MAC="); Serial.println(mac_00);

    // B[0][p]를 오른쪽(PE(0,1))으로 전송 — PE(0,1)의 C[0][p] 계산용
    serRight.println(b0);

    // B[0][p]와 B[1][p] 모두를 아래(PE(1,0))로 전송 — C[1][p] 계산용
    serDown.println(b0);  // PE(1,0)이 A[1][0]*B[0][p] 계산에 사용
    serDown.println(b1);  // PE(1,0)이 PE(1,1)으로 전달

    delay(50); // 다음 열 처리 전 안정화
  }

  digitalWrite(PIN_D6, LOW);

  Serial.println("[PE(0,0)] Done. Press reset to re-run.");
  while (true) {}
}
```

### PE(0,1) — 오른쪽 상단, C[0][p] 계산 종단

PE(0,1)은 PE(0,0)으로부터 B[0][p]를 수신하고, 자신의 고정 가중치 W[0][1]=3과 곱해 C[0][p]의 두 번째 항을 계산합니다. 이 부분합은 C[0][p]에만 속하므로 PE(1,1)으로 전달하지 않습니다. (PE(1,1)에 전달하면 C[1][p] 계산이 틀립니다.)

```cpp
// WS 모드 — PE(0,1)
// 고정 가중치: W[0][1] = 3 = A[0][1]
// PE(0,0)으로부터 B[0][p] 수신 → 자신의 MAC 연산 (C[0][p]의 두 번째 항)
// 이 보드는 C[0][p] 계산의 종단점이며, 결과를 PE(1,1)로 전달하지 않습니다.
// (PE(0,1)의 부분합은 C[0][p]에 속하며 C[1][p]와는 무관합니다.)

#include <SoftwareSerial.h>

// 왼쪽(PE(0,0))에서 수신
SoftwareSerial serLeft(2, 3);  // RX=D2, TX=D3

// 고정 가중치
const int W = 3;  // W[0][1] = A[0][1]

#define P_SIZE 2   // B의 열 수

void setup() {
  Serial.begin(9600);
  serLeft.begin(9600);

  delay(2000);
  Serial.println("[PE(0,1)] WS mode ready.");
}

void loop() {
  for (int p = 0; p < P_SIZE; p++) {
    // B[0][p] 수신 (PE(0,0)이 보낸 값)
    while (!serLeft.available()) {}
    int b0 = serLeft.parseInt();

    // MAC: W[0][1] × B[0][p] — C[0][p]의 두 번째 항
    int partialSum = W * b0;
    Serial.print("[PE(0,1)] p="); Serial.print(p);
    Serial.print(" b0="); Serial.print(b0);
    Serial.print(" W*b0="); Serial.println(partialSum);

    delay(10);
  }

  // PE(0,1)의 partialSum은 C[0][p] = A[0][0]*B[0][p] + A[0][1]*B[0][p] 중
  // 두 번째 항 A[0][1]*B[0][p]에 해당합니다.
  // 첫 번째 항 A[0][0]*B[0][p]는 PE(0,0)이 자체적으로 계산합니다.
  // 이 보드의 시리얼을 노트북에 연결하면 위 로그로 중간값 검증이 가능합니다.

  Serial.println("[PE(0,1)] Done. Press reset to re-run.");
  while (true) {}
}
```

### PE(1,0) — 왼쪽 하단, 부분합과 B[1][p]를 오른쪽으로 전송

PE(1,0)은 PE(0,0)으로부터 B[0][p]와 B[1][p]를 순서대로 수신합니다. 자신의 가중치 W[1][0]=4와 B[0][p]를 곱해 C[1][p]의 첫 번째 항을 계산하고, 그 부분합과 B[1][p]를 연속으로 PE(1,1)에 전송합니다. PE(1,1)은 이 두 값으로 나머지 항을 완성합니다.

```cpp
// WS 모드 — PE(1,0)
// 고정 가중치: W[1][0] = 4 = A[1][0]
// PE(0,0)으로부터 B[0][p]와 B[1][p]를 순서대로 수신
// 자신의 MAC: A[1][0] × B[0][p] (C[1][p]의 첫 번째 항)
// PE(1,1)으로 부분합(4×B[0][p])과 B[1][p]를 전송
// PE(1,1)은 부분합 + A[1][1]×B[1][p]로 C[1][p]를 완성

#include <SoftwareSerial.h>

// 위(PE(0,0))에서 수신
SoftwareSerial serUp(2, 3);    // RX=D2, TX=D3

// 오른쪽(PE(1,1))으로 전송
SoftwareSerial serRight(4, 5); // RX=D4, TX=D5

// 고정 가중치
const int W = 4;  // W[1][0] = A[1][0]

#define P_SIZE 2   // B의 열 수

void setup() {
  Serial.begin(9600);
  serUp.begin(9600);
  serRight.begin(9600);

  delay(2000);
  Serial.println("[PE(1,0)] WS mode ready.");
}

void loop() {
  for (int p = 0; p < P_SIZE; p++) {
    // B[0][p] 수신 — C[1][p]의 첫 번째 항에 사용
    while (!serUp.available()) {}
    int b0 = serUp.parseInt();

    // B[1][p] 수신 — PE(1,1)으로 전달
    while (!serUp.available()) {}
    int b1 = serUp.parseInt();

    // MAC: W[1][0] × B[0][p] (C[1][p]의 첫 번째 항)
    int partialSum = W * b0;
    Serial.print("[PE(1,0)] p="); Serial.print(p);
    Serial.print(" b0="); Serial.print(b0);
    Serial.print(" b1="); Serial.print(b1);
    Serial.print(" partialSum="); Serial.println(partialSum);

    // 부분합(4×B[0][p])과 B[1][p]를 PE(1,1)로 전송
    serRight.println(partialSum);
    serRight.println(b1);

    delay(10);
  }

  Serial.println("[PE(1,0)] Done. Press reset to re-run.");
  while (true) {}
}
```

### PE(1,1) — 최종 누산기, LCD 출력

PE(1,1)은 PE(1,0)으로부터 부분합(`4×B[0][p]`)과 `B[1][p]` 값을 수신합니다. 자신의 고정 가중치 W[1][1]=5와 `B[1][p]`를 곱한 후 부분합에 더해 C[1][p]를 완성합니다. D6 신호로 처리 시작 시각을 기록하고 결과와 경과 시간을 LCD에 표시합니다.

계산 검증 (p=1):
- `ps_left` = PE(1,0)이 보낸 부분합 = 4 × B[0][1] = 4 × 2 = **8**
- `b1` = PE(1,0)이 전달한 B[1][1] = **4**
- `myMAC` = W × b1 = 5 × 4 = **20**
- C[1][1] = 8 + 20 = **28** ✓

```cpp
// WS 모드 — PE(1,1)
// 고정 가중치: W[1][1] = 5 = A[1][1]
// PE(1,0)으로부터 부분합(A[1][0]*B[0][p])과 B[1][p] 수신
// 자신의 MAC: A[1][1] × B[1][p]
// C[1][p] = 부분합 + 자신의 MAC = A[1][0]*B[0][p] + A[1][1]*B[1][p] ✓

#include <SoftwareSerial.h>
#include <Wire.h>
#include <LiquidCrystal_I2C.h>

// 왼쪽(PE(1,0))에서 수신
SoftwareSerial serLeft(2, 3);  // RX=D2, TX=D3

#define PIN_D6 6   // 타이밍 신호 입력

// I2C LCD (주소 0x27, 16열 2행)
LiquidCrystal_I2C lcd(0x27, 16, 2);

// 고정 가중치
const int W = 5;  // W[1][1] = A[1][1]

#define P_SIZE 2   // B의 열 수

unsigned long startTime = 0;
unsigned long endTime = 0;

void setup() {
  Serial.begin(9600);
  serLeft.begin(9600);

  pinMode(PIN_D6, INPUT);

  lcd.init();
  lcd.backlight();
  lcd.setCursor(0, 0);
  lcd.print("WS mode ready");
  lcd.setCursor(0, 1);
  lcd.print("Waiting D6...");

  delay(2000);
  Serial.println("[PE(1,1)] WS mode ready.");
}

void loop() {
  int finalResult_p0 = 0;  // C[1][0]
  int finalResult_p1 = 0;  // C[1][1] (LCD 표시값)

  // D6 HIGH 대기: 타이밍 측정 시작
  while (digitalRead(PIN_D6) == LOW) {}
  startTime = micros();
  Serial.println("[PE(1,1)] D6 HIGH detected. Timing started.");

  // --- p=0 열 처리: C[1][0] 계산 ---
  // PE(1,0)으로부터 부분합(A[1][0]*B[0][0]) 수신
  while (!serLeft.available()) {}
  int ps_left_0 = serLeft.parseInt();

  // PE(1,0)이 이어서 전달한 B[1][0] 수신
  while (!serLeft.available()) {}
  int b1_p0 = serLeft.parseInt();

  // 자신의 MAC: A[1][1] × B[1][0]
  int myMAC_0 = W * b1_p0;
  finalResult_p0 = ps_left_0 + myMAC_0;
  Serial.print("[PE(1,1)] C[1][0]="); Serial.println(finalResult_p0);
  // 기댓값: 4*1 + 5*3 = 4+15 = 19

  // --- p=1 열 처리: C[1][1] 계산 ---
  // PE(1,0)으로부터 부분합(A[1][0]*B[0][1]) 수신
  while (!serLeft.available()) {}
  int ps_left_1 = serLeft.parseInt();

  // PE(1,0)이 이어서 전달한 B[1][1] 수신
  while (!serLeft.available()) {}
  int b1_p1 = serLeft.parseInt();

  // 자신의 MAC: A[1][1] × B[1][1]
  int myMAC_1 = W * b1_p1;

  // 최종 MAC 완료 시점 기록
  endTime = micros();

  finalResult_p1 = ps_left_1 + myMAC_1;
  Serial.print("[PE(1,1)] C[1][1]="); Serial.println(finalResult_p1);
  // 기댓값: 4*2 + 5*4 = 8+20 = 28

  unsigned long elapsed = endTime - startTime;
  Serial.print("[PE(1,1)] Elapsed="); Serial.print(elapsed); Serial.println(" us");

  // LCD 출력
  lcd.clear();
  lcd.setCursor(0, 0);
  lcd.print("C[1][1]=");
  lcd.print(finalResult_p1);

  lcd.setCursor(0, 1);
  lcd.print("Time=");
  lcd.print(elapsed);
  lcd.print("us");

  Serial.println("[PE(1,1)] Done. Press reset to re-run.");
  while (true) {}
}
```

## OS 모드에서 WS 모드로 전환하기

두 모드의 전환은 코드 재업로드만으로 완료됩니다. 하드웨어 배선은 동일합니다.

<Steps>
  <Step title="WS 모드 코드 4개 준비">
    각 PE에 해당하는 WS 모드 `.ino` 파일을 Arduino IDE에서 엽니다. 파일명 예시: `ws_pe00.ino`, `ws_pe01.ino`, `ws_pe10.ino`, `ws_pe11.ino`
  </Step>
  <Step title="보드를 노트북에 순서대로 연결">
    USB 허브 대신 노트북에 직접 연결하여 업로드합니다. 업로드 순서: PE(1,1) → PE(1,0) → PE(0,1) → PE(0,0). 마스터인 PE(0,0)을 마지막에 업로드해야 초기화 순서가 맞습니다.
  </Step>
  <Step title="가중치 상수 확인">
    각 보드 코드 상단의 `const int W = ...;` 값이 A 행렬의 해당 원소와 일치하는지 확인합니다.
    - PE(0,0): `W = 2`, PE(0,1): `W = 3`, PE(1,0): `W = 4`, PE(1,1): `W = 5`
  </Step>
  <Step title="행렬 크기 #define 맞추기">
    측정 조건에 따라 `#define M_SIZE`, `#define N_SIZE`, `#define P_SIZE` 값을 수정합니다. 4개 보드 모두 동일한 값으로 설정해야 합니다.
  </Step>
  <Step title="업로드 및 전원 인가">
    모든 보드 업로드 완료 후 USB 허브에 연결합니다. PE(0,0)이 2초 대기 후 자동으로 전송을 시작합니다.
  </Step>
</Steps>

<Tip>
행렬 크기를 변경할 때는 4개 보드의 `#define` 값을 동시에 수정하고 모두 재업로드해야 합니다. 한 보드만 변경하면 타이밍 불일치로 데이터가 손실됩니다.
</Tip>

## WS 모드에서 달라지는 점 요약

| 변경 사항 | OS 모드 | WS 모드 |
|-----------|---------|---------|
| PE(0,0)이 전송하는 값 | A 행 + B 열 | B 행 값 쌍(B[0][p], B[1][p]) |
| 각 PE의 가중치 처리 | 수신 후 저장 | 펌웨어 상수 |
| PE(1,1)의 수신 경로 | PE(1,0) + PE(0,1) (양쪽) | PE(1,0)만 (부분합 + B[1][p]) |
| 통신 패킷 수 | 많음 | 중간 (올바른 라우팅 필요) |
| 코드 복잡도 | 상대적으로 높음 | 낮음 |
| 결과 확인 위치 | LCD (PE(1,1)) | LCD (PE(1,1)) |
