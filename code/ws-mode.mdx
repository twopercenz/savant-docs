WS(Weight Stationary) 모드는 각 PE(Processing Element)가 자신이 담당하는 가중치 행렬 원소를 펌웨어에 하드코딩해 고정하고, 오직 입력값(B 행렬 원소)만 격자 사이를 흘려보내는 방식입니다. OS 모드에서는 A·B·부분합 세 종류의 값이 이동하지만, WS 모드에서는 가중치를 전송할 필요가 없으므로 통신 횟수가 줄어들고, 각 PE의 역할이 단순해집니다. 아래에서 WS 모드의 동작 원리, 보드별 역할, 전체 코드, 그리고 OS 모드에서 전환하는 방법을 순서대로 설명합니다.

## WS 모드 vs OS 모드: 핵심 차이

| 항목 | OS 모드 | WS 모드 |
|------|---------|---------|
| 가중치(A) 전송 | PE 간 전달 | 각 PE에 고정(하드코딩) |
| 입력(B) 전송 | PE 간 전달 | PE 간 전달 |
| 부분합 전송 | PE 간 전달 | PE 간 전달 |
| 통신 데이터 종류 | A, B, 부분합 | B, 부분합 |
| 모드 전환 방법 | 다른 코드 세트 업로드 | 다른 코드 세트 업로드 |

<Note>
WS 모드에서는 PE(0,0)이 B 열의 값만 오른쪽·아래로 전송합니다. A 행렬의 원소는 전송되지 않고, 각 PE의 펌웨어에 상수로 고정됩니다.
</Note>

## 보드별 역할

기본값: A = [[2, 3], [4, 5]], B = [[1, 2], [3, 4]], 기댓값 C = [[11, 16], [19, 28]]

| 보드 | 고정 가중치 | 받는 값 | 보내는 값 |
|------|-----------|---------|----------|
| PE(0,0) | W[0][0] = 2 | 없음 (마스터) | B 열 입력 → PE(0,1) (오른쪽), PE(1,0) (아래) |
| PE(0,1) | W[0][1] = 3 | 입력 ← PE(0,0) | 부분합 → PE(1,1) (아래) |
| PE(1,0) | W[1][0] = 4 | 입력 ← PE(0,0) | 부분합 → PE(1,1) (오른쪽) |
| PE(1,1) | W[1][1] = 5 | 입력 ← PE(0,0), 부분합 ← PE(0,1), PE(1,0) | 최종 C[1][1] LCD 출력 |

<Note>
C[0][p] 결과는 PE(0,1)이 자체적으로 계산합니다. 해당 값은 LCD에는 표시되지 않으며, PE(0,1)의 시리얼 포트를 순간적으로 노트북에 연결하면 시리얼 모니터로 확인할 수 있습니다.
</Note>

## 핀 배치 (전 보드 공통)

| 핀 | 역할 |
|----|------|
| D2 (RX_SW) | SoftwareSerial 수신 |
| D3 (TX_SW) | SoftwareSerial 송신 |
| D4 | 추가 수신 채널 (PE(1,1) 전용) |
| D5 | 추가 송신 채널 (PE(1,1) 전용) |
| D6 | 타이밍 신호 (PE(0,0) 출력 / PE(1,1) 입력) |
| A4 (SDA) | I2C LCD — PE(1,1) 전용 |
| A5 (SCL) | I2C LCD — PE(1,1) 전용 |

## 전체 코드

### PE(0,0) — 마스터, D6 타이밍 신호 출력

PE(0,0)은 B 행렬의 열 값을 하나씩 오른쪽(PE(0,1))과 아래(PE(1,0))로 전송합니다. 자신의 MAC 연산 직전에 D6를 HIGH로 설정하여 타이밍 측정을 시작합니다.

```cpp
// WS 모드 — PE(0,0)
// 고정 가중치: W[0][0] = 2
// B 열 입력을 PE(0,1)과 PE(1,0)으로 전송

#include <SoftwareSerial.h>

// SoftwareSerial: 오른쪽(PE(0,1))으로 전송
SoftwareSerial serRight(2, 3); // RX=D2, TX=D3

// 아래(PE(1,0))로 전송하는 추가 시리얼
SoftwareSerial serDown(4, 5);  // RX=D4, TX=D5

#define PIN_D6 6   // 타이밍 신호 출력

// 고정 가중치
const int W = 2;  // W[0][0]

// B 행렬 (2×2 기준 — 열별로 PE(0,0)이 관리)
// B = [[1,2],[3,4]]
// PE(0,0)은 B[0][0], B[1][0] (첫 번째 열) 을 처리
const int B_col0[] = {1, 3};   // B 1열: B[0][0], B[1][0]
const int B_col1[] = {2, 4};   // B 2열: B[0][1], B[1][1]

// 행렬 크기 설정 (#define으로 조건 전환)
#define M_SIZE 2   // A의 행 수
#define P_SIZE 2   // B의 열 수
#define N_SIZE 2   // A의 열 수 = B의 행 수

void setup() {
  Serial.begin(9600);
  serRight.begin(9600);
  serDown.begin(9600);

  pinMode(PIN_D6, OUTPUT);
  digitalWrite(PIN_D6, LOW);

  delay(2000); // 모든 보드 초기화 대기
  Serial.println("[PE(0,0)] WS mode ready.");
}

void loop() {
  // 2×2×2 기준: p=0, p=1 두 열 처리
  for (int p = 0; p < P_SIZE; p++) {
    // 현재 열의 B 입력값 선택
    const int* B_col = (p == 0) ? B_col0 : B_col1;

    // --- 첫 번째 타임스텝: B[0][p] 전송 ---
    // 자신의 MAC 직전에 D6 HIGH (타이밍 측정 시작)
    digitalWrite(PIN_D6, HIGH);

    // PE(0,0) 자신의 MAC: W[0][0] × B[0][p]
    int mac_00 = W * B_col[0];
    Serial.print("[PE(0,0)] p="); Serial.print(p);
    Serial.print(" MAC="); Serial.println(mac_00);

    // B[0][p]를 오른쪽(PE(0,1))으로 전송
    serRight.println(B_col[0]);

    // --- 두 번째 타임스텝: B[1][p] 전송 ---
    // B[1][p]를 아래(PE(1,0))로 전송
    serDown.println(B_col[1]);

    delay(50); // 다음 열 처리 전 안정화
  }

  digitalWrite(PIN_D6, LOW);

  // 한 번만 실행 후 대기 (리셋으로 재실행)
  Serial.println("[PE(0,0)] Done. Press reset to re-run.");
  while (true) {}
}
```

### PE(0,1) — 오른쪽 상단, 부분합 아래로 전송

PE(0,1)은 PE(0,0)으로부터 B의 행 값을 수신하고, 자신의 고정 가중치 W[0][1]=3과 곱하여 누적합산한 뒤 부분합을 PE(1,1)로 전송합니다.

```cpp
// WS 모드 — PE(0,1)
// 고정 가중치: W[0][1] = 3
// PE(0,0)으로부터 입력 수신 → 부분합 PE(1,1)로 전송

#include <SoftwareSerial.h>

// 왼쪽(PE(0,0))에서 수신
SoftwareSerial serLeft(2, 3);  // RX=D2, TX=D3

// 아래(PE(1,1))로 전송
SoftwareSerial serDown(4, 5);  // RX=D4, TX=D5

// 고정 가중치
const int W = 3;  // W[0][1]

#define P_SIZE 2   // B의 열 수

void setup() {
  Serial.begin(9600);
  serLeft.begin(9600);
  serDown.begin(9600);

  delay(2000);
  Serial.println("[PE(0,1)] WS mode ready.");
}

void loop() {
  for (int p = 0; p < P_SIZE; p++) {
    int partialSum = 0;

    // B[0][p] 수신 (PE(0,0)이 보낸 값)
    while (!serLeft.available()) {}
    int b0 = serLeft.parseInt();

    // MAC: W[0][1] × B[0][p]
    partialSum = W * b0;
    Serial.print("[PE(0,1)] p="); Serial.print(p);
    Serial.print(" b0="); Serial.print(b0);
    Serial.print(" partialSum="); Serial.println(partialSum);

    // 부분합을 PE(1,1)로 전송
    serDown.println(partialSum);

    delay(10);
  }

  // C[0][p] 결과 출력 (시리얼 연결 시 확인용)
  // C[0][0] = W[0][0]*B[0][0] + W[0][1]*B[1][0] — PE(0,0)의 MAC + 자신의 MAC
  // 참고: PE(0,1)의 partialSum만으로는 C[0][p] 전체가 아님
  // 이 보드의 시리얼을 연결하면 위 로그로 값 검증 가능

  Serial.println("[PE(0,1)] Done. Press reset to re-run.");
  while (true) {}
}
```

### PE(1,0) — 왼쪽 하단, 부분합 오른쪽으로 전송

PE(1,0)은 PE(0,0)으로부터 B의 행 값을 수신하고, 자신의 고정 가중치 W[1][0]=4와 곱하여 부분합을 PE(1,1)로 전송합니다.

```cpp
// WS 모드 — PE(1,0)
// 고정 가중치: W[1][0] = 4
// PE(0,0)으로부터 입력 수신 → 부분합 PE(1,1)로 전송

#include <SoftwareSerial.h>

// 위(PE(0,0))에서 수신
SoftwareSerial serUp(2, 3);    // RX=D2, TX=D3

// 오른쪽(PE(1,1))으로 전송
SoftwareSerial serRight(4, 5); // RX=D4, TX=D5

// 고정 가중치
const int W = 4;  // W[1][0]

#define P_SIZE 2   // B의 열 수

void setup() {
  Serial.begin(9600);
  serUp.begin(9600);
  serRight.begin(9600);

  delay(2000);
  Serial.println("[PE(1,0)] WS mode ready.");
}

void loop() {
  for (int p = 0; p < P_SIZE; p++) {
    int partialSum = 0;

    // B[1][p] 수신 (PE(0,0)이 보낸 값)
    while (!serUp.available()) {}
    int b1 = serUp.parseInt();

    // MAC: W[1][0] × B[1][p]
    partialSum = W * b1;
    Serial.print("[PE(1,0)] p="); Serial.print(p);
    Serial.print(" b1="); Serial.print(b1);
    Serial.print(" partialSum="); Serial.println(partialSum);

    // 부분합을 PE(1,1)로 전송
    serRight.println(partialSum);

    delay(10);
  }

  Serial.println("[PE(1,0)] Done. Press reset to re-run.");
  while (true) {}
}
```

### PE(1,1) — 최종 누산기, LCD 출력

PE(1,1)은 PE(0,1)로부터 위쪽 부분합, PE(1,0)으로부터 왼쪽 부분합을 수신하고, 자신의 고정 가중치 W[1][1]=5와 입력을 곱한 값을 누적하여 최종 C[1][1]를 계산합니다. D6 신호로 처리 시작 시각을 기록하고 결과와 경과 시간을 LCD에 표시합니다.

```cpp
// WS 모드 — PE(1,1)
// 고정 가중치: W[1][1] = 5
// PE(0,1)(위), PE(1,0)(왼쪽)에서 부분합 수신 → 최종 MAC → LCD 출력

#include <SoftwareSerial.h>
#include <Wire.h>
#include <LiquidCrystal_I2C.h>

// 위(PE(0,1))에서 수신
SoftwareSerial serUp(2, 3);    // RX=D2, TX=D3

// 왼쪽(PE(1,0))에서 수신
SoftwareSerial serLeft(4, 5);  // RX=D4, TX=D5

#define PIN_D6 6   // 타이밍 신호 입력

// I2C LCD (주소 0x27, 16열 2행)
LiquidCrystal_I2C lcd(0x27, 16, 2);

// 고정 가중치
const int W = 5;  // W[1][1]

#define P_SIZE 2   // B의 열 수

// 타이밍
unsigned long startTime = 0;
unsigned long endTime = 0;
bool timingStarted = false;

void setup() {
  Serial.begin(9600);
  serUp.begin(9600);
  serLeft.begin(9600);

  pinMode(PIN_D6, INPUT);

  lcd.init();
  lcd.backlight();
  lcd.setCursor(0, 0);
  lcd.print("WS mode ready");
  lcd.setCursor(0, 1);
  lcd.print("Waiting D6...");

  delay(2000);
  Serial.println("[PE(1,1)] WS mode ready.");
}

void loop() {
  int finalResult_p0 = 0;  // C[1][0]
  int finalResult_p1 = 0;  // C[1][1] (LCD 표시값)

  // --- D6 HIGH 대기: 타이밍 측정 시작 ---
  while (digitalRead(PIN_D6) == LOW) {}
  startTime = micros();
  timingStarted = true;
  Serial.println("[PE(1,1)] D6 HIGH detected. Timing started.");

  // --- p=0 열 처리 ---
  // PE(0,1)로부터 부분합 수신 (W[0][1]*B[0][0])
  while (!serUp.available()) {}
  int ps_up_0 = serUp.parseInt();

  // PE(1,0)로부터 부분합 수신 (W[1][0]*B[1][0])
  while (!serLeft.available()) {}
  int ps_left_0 = serLeft.parseInt();

  // 자신의 MAC: W[1][1] × B[0][0]
  // B[0][0]은 직접 받지 않으므로 PE(0,0)이 PE(1,1)에게도 전달해야 하는 경우
  // WS 모드 단순화를 위해 PE(1,1)은 부분합만 누산하는 구조로 구현
  // C[1][0] = ps_up_0 + ps_left_0 + W*B[0][0]
  // 여기서는 부분합 두 개를 합산하고 자신의 MAC을 위해 B 값을 PE(0,0)이 직접 전송
  // (아래 주석 참고: 실제 배선에서는 PE(0,0)이 D4/D5로도 B 값을 PE(1,1)에 전송)

  // 단순 구현: 부분합 합산 (W*B 항은 PE(0,1), PE(1,0)이 각각 W*B[0][p], W*B[1][p] 처리)
  // C[1][0] = ps_up_0(W[0][1]*B[0][0]) + ps_left_0(W[1][0]*B[1][0])
  // + PE(1,1) 자신: W[1][1]*B[0][0] — B[0][0]은 별도 수신 필요
  // 여기서는 실험 구성에 따라 PE(0,0)이 PE(1,1)에도 B 입력을 전달하는 방식 사용:

  // 실제 WS 2×2 구현에서 C[1][p] = sum_k(A[1][k]*B[k][p])
  //   = A[1][0]*B[0][p] + A[1][1]*B[1][p]
  //   = ps_left_0 (PE(1,0)이 A[1][0]*B[0][p] 계산)
  //   + W * B[1][p]     (PE(1,1) 자신이 A[1][1]*B[1][p] 계산)
  // 따라서 PE(1,1)은 PE(1,0)의 부분합 + 자신의 MAC이 필요
  // B[1][p]는 PE(1,0)을 경유하거나 PE(0,0)이 직접 전송

  // 아래 코드는 PE(1,0)이 B[1][p]도 함께 전송하는 방식으로 구현
  // (직렬 패킷: "부분합,B값" 형태)
  // 편의상 이 예제에서는 부분합만 수신하고 B[1][p]는 0으로 가정하지 않음
  // 대신 PE(1,0)이 partialSum = W[1][0]*B[1][p] 를 보내므로
  // PE(1,1)은 추가로 자신의 W[1][1]*B[1][p] 를 계산하기 위해 B[1][p]가 필요

  // --- 실용적 구현: PE(1,1)의 자체 MAC을 위한 B[1][p] 수신 ---
  // PE(0,0)이 B[1][0], B[1][1]을 추가 채널(D4/D5)로 PE(1,1)에 직접 전송하는 방식 가정
  // 아래에서는 pe00_b1이라는 변수로 처리
  // (실험 보드 배선에서 PE(0,0)의 serExtra TX → PE(1,1)의 serExtra RX)

  // 이 예제에서는 부분합만 받아 누산하는 단순 구조 사용:
  // PE(0,1)이 W[0][1]*B[0][p] 전송 (위에서 수신한 ps_up_0)
  // PE(1,0)이 W[1][0]*B[1][p] 전송 (왼쪽에서 수신한 ps_left_0)
  // PE(1,1) 자신: W[1][1]*B[1][p] — B[1][p]를 별도 수신
  // 간소화: B[1][p]를 serLeft로 받는 두 번째 parseInt 사용

  int b1_p0 = serLeft.parseInt();  // PE(1,0)이 이어서 B[1][0]를 전송
  int myMAC_0 = W * b1_p0;

  finalResult_p0 = ps_up_0 + ps_left_0 + myMAC_0;
  Serial.print("[PE(1,1)] C[1][0]="); Serial.println(finalResult_p0);

  // --- p=1 열 처리 ---
  while (!serUp.available()) {}
  int ps_up_1 = serUp.parseInt();

  while (!serLeft.available()) {}
  int ps_left_1 = serLeft.parseInt();

  int b1_p1 = serLeft.parseInt();
  int myMAC_1 = W * b1_p1;

  // 최종 MAC 완료 시점 기록
  endTime = micros();

  finalResult_p1 = ps_up_1 + ps_left_1 + myMAC_1;
  Serial.print("[PE(1,1)] C[1][1]="); Serial.println(finalResult_p1);

  unsigned long elapsed = endTime - startTime;
  Serial.print("[PE(1,1)] Elapsed="); Serial.print(elapsed); Serial.println(" us");

  // LCD 출력
  lcd.clear();
  lcd.setCursor(0, 0);
  lcd.print("C[1][1]=");
  lcd.print(finalResult_p1);

  lcd.setCursor(0, 1);
  lcd.print("Time=");
  lcd.print(elapsed);
  lcd.print("us");

  Serial.println("[PE(1,1)] Done. Press reset to re-run.");
  while (true) {}
}
```

<Warning>
PE(1,1)의 B[1][p] 수신 방식은 실험 배선에 따라 달라질 수 있습니다. 위 코드는 PE(1,0)이 부분합 전송 후 연속으로 B[1][p] 값을 추가 전송하는 방식을 가정합니다. 실제 배선에서 채널이 부족할 경우, PE(0,0)이 별도 채널로 PE(1,1)에 직접 B[1][p]를 전송하도록 수정하십시오.
</Warning>

## OS 모드에서 WS 모드로 전환하기

두 모드의 전환은 코드 재업로드만으로 완료됩니다. 하드웨어 배선은 동일합니다.

<Steps>
  <Step title="WS 모드 코드 4개 준비">
    각 PE에 해당하는 WS 모드 `.ino` 파일을 Arduino IDE에서 엽니다. 파일명 예시: `ws_pe00.ino`, `ws_pe01.ino`, `ws_pe10.ino`, `ws_pe11.ino`
  </Step>
  <Step title="보드를 노트북에 순서대로 연결">
    USB 허브 대신 노트북에 직접 연결하여 업로드합니다. 업로드 순서: PE(1,1) → PE(1,0) → PE(0,1) → PE(0,0). 마스터인 PE(0,0)을 마지막에 업로드해야 초기화 순서가 맞습니다.
  </Step>
  <Step title="가중치 상수 확인">
    각 보드 코드 상단의 `const int W = ...;` 값이 A 행렬의 해당 원소와 일치하는지 확인합니다.
    - PE(0,0): `W = 2`, PE(0,1): `W = 3`, PE(1,0): `W = 4`, PE(1,1): `W = 5`
  </Step>
  <Step title="행렬 크기 #define 맞추기">
    측정 조건에 따라 `#define M_SIZE`, `#define N_SIZE`, `#define P_SIZE` 값을 수정합니다. 4개 보드 모두 동일한 값으로 설정해야 합니다.
  </Step>
  <Step title="업로드 및 전원 인가">
    모든 보드 업로드 완료 후 USB 허브에 연결합니다. PE(0,0)이 2초 대기 후 자동으로 전송을 시작합니다.
  </Step>
</Steps>

<Tip>
행렬 크기를 변경할 때는 4개 보드의 `#define` 값을 동시에 수정하고 모두 재업로드해야 합니다. 한 보드만 변경하면 타이밍 불일치로 데이터가 손실됩니다.
</Tip>

## WS 모드에서 달라지는 점 요약

| 변경 사항 | OS 모드 | WS 모드 |
|-----------|---------|---------|
| PE(0,0)이 전송하는 값 | A 행 + B 열 | B 열만 |
| 각 PE의 가중치 처리 | 수신 후 저장 | 펌웨어 상수 |
| 통신 패킷 수 | 많음 | 적음 |
| 코드 복잡도 | 상대적으로 높음 | 낮음 |
| 결과 확인 위치 | LCD (PE(1,1)) | LCD (PE(1,1)) |
