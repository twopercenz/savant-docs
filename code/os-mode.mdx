OS(Output Stationary) 모드는 시스톨릭 어레이의 각 PE(Processing Element)가 **자신의 출력 원소를 고정한 채** 누산하는 방식입니다. 2×2 격자에서 A 행렬의 값은 왼쪽→오른쪽으로, B 행렬의 값은 위→아래 방향으로 흘러가며, 각 PE는 도착한 값으로 MAC(Multiply-Accumulate) 연산을 수행합니다. 4개의 Arduino Nano가 각각 PE(0,0), PE(0,1), PE(1,0), PE(1,1) 역할을 맡아 커스텀 클럭-동기 통신(`shiftOut`/`receiveByte`)으로 데이터를 주고받습니다.

## 격자 구조와 데이터 흐름

```
         B[0][0]=1      B[0][1]=2
              ↓              ↓
A[0][*] → PE(0,0) —A→ PE(0,1)
              ↓B             ↓B
A[1][*] → PE(1,0) —A→ PE(1,1) ← TIMING
```

| PE 위치 | 담당 원소 | 소유 A값 | 소유 B값 | A 방향 | B 방향 |
|---------|----------|---------|---------|--------|--------|
| PE(0,0) | C[0][0]  | A[0][0]=2 | B[0][0]=1 | → PE(0,1) 전송 | ↓ PE(1,0) 전송 |
| PE(0,1) | C[0][1]  | 수신      | B[0][1]=2 | —      | ↓ PE(1,1) 전송 |
| PE(1,0) | C[1][0]  | A[1][0]=4 | 수신      | → PE(1,1) 전송 | —      |
| PE(1,1) | C[1][1]  | 수신      | 수신      | —      | —      |

<Note>
이 모드에서 각 PE는 하나의 출력 원소(C 행렬의 한 칸)만 계산합니다. 2×2 행렬 곱셈의 경우 각 PE의 MAC 연산은 단 1회이며, 타이밍 측정은 PE(0,0)의 MAC 시작부터 PE(1,1)의 MAC 완료까지를 기준으로 합니다.
</Note>

## 핀 배치 요약

모든 보드가 동일한 물리 핀 번호를 사용하지만, 각 보드에서 해당 핀이 어디로 연결되는지가 다릅니다.

| 핀 번호 | 신호 종류 | 방향 |
|---------|----------|------|
| D2 | CLOCK 라인 1 | 보드별 상이 |
| D3 | CLOCK 라인 2 | 보드별 상이 |
| D4 | DATA 라인 1  | 보드별 상이 |
| D5 | DATA 라인 2  | 보드별 상이 |
| D6 | 타이밍 신호   | PE(0,0) → PE(1,1) |

## PE(0,0) 코드

PE(0,0)은 격자의 시작점으로, A[0][0]과 B[0][0]을 모두 소유합니다. 자신의 MAC 연산을 가장 먼저 수행하고 타이밍 신호를 D6으로 보낸 뒤, A값은 오른쪽의 PE(0,1)으로, B값은 아래쪽의 PE(1,0)으로 전송합니다.

### 역할 요약

- **계산**: C[0][0] = A[0][0] × B[0][0] = 2 × 1 = **2** (2×2 행렬에서 단일 항)
- **송신**: A[0][0]=2 → PE(0,1) / B[0][0]=1 → PE(1,0)
- **타이밍**: MAC 직전 D6 HIGH → PE(1,1)이 `startTime` 기록

### 핀 배치

| 핀 | 상수 | 방향 | 대상 |
|----|------|------|------|
| D4 | `DATA_A_TX`  | OUTPUT | PE(0,1) DATA |
| D2 | `CLOCK_A_TX` | OUTPUT | PE(0,1) CLOCK |
| D5 | `DATA_B_TX`  | OUTPUT | PE(1,0) DATA |
| D3 | `CLOCK_B_TX` | OUTPUT | PE(1,0) CLOCK |
| D6 | `TIMING_PIN` | OUTPUT | PE(1,1) TIMING_RX |

### 전체 코드

```cpp
// pe00_os.ino — PE(0,0) Output Stationary 모드
// 역할: 격자의 시작점. 자신의 MAC 수행 후 A값→오른쪽, B값→아래쪽 전송

#define DATA_A_TX 4   // A값 데이터 → PE(0,1)
#define CLOCK_A_TX 2  // A값 클럭  → PE(0,1)
#define DATA_B_TX 5   // B값 데이터 → PE(1,0)
#define CLOCK_B_TX 3  // B값 클럭  → PE(1,0)
#define TIMING_PIN 6  // 타이밍 신호 → PE(1,1)

int A00 = 2;  // A[0][0]
int B00 = 1;  // B[0][0]
int localSum = 0;

void setup() {
  pinMode(DATA_A_TX, OUTPUT);
  pinMode(CLOCK_A_TX, OUTPUT);
  pinMode(DATA_B_TX, OUTPUT);
  pinMode(CLOCK_B_TX, OUTPUT);
  pinMode(TIMING_PIN, OUTPUT);
  digitalWrite(TIMING_PIN, LOW);

  delay(500);  // 다른 보드 초기화 대기

  // 타이밍 신호 시작 (MAC 시작 직전)
  digitalWrite(TIMING_PIN, HIGH);

  // 자신의 MAC 연산
  localSum += A00 * B00;

  // A값을 오른쪽(PE(0,1))으로 전송
  shiftOut(DATA_A_TX, CLOCK_A_TX, MSBFIRST, (byte)A00);

  // B값을 아래쪽(PE(1,0))으로 전송
  shiftOut(DATA_B_TX, CLOCK_B_TX, MSBFIRST, (byte)B00);
}

void loop() {}
```

### 행렬 값 변경 방법

`A00`과 `B00` 변수를 수정하세요. 예를 들어 A[0][0]=3, B[0][0]=2로 바꾸려면:

```cpp
int A00 = 3;  // A[0][0] 변경
int B00 = 2;  // B[0][0] 변경
```

<Warning>
`delay(500)`은 나머지 3개 보드가 먼저 수신 대기 상태에 들어갈 시간을 확보합니다. PE(0,0)을 **마지막**에 업로드하거나 리셋하는 이유가 바로 이 때문입니다. 이 딜레이를 줄이면 수신 보드가 준비되기 전에 신호가 전송될 수 있습니다.
</Warning>

## PE(0,1) 코드

PE(0,1)은 A값을 PE(0,0)으로부터 수신하고, B[0][1]=2는 자신이 소유합니다. MAC 연산 후 B값을 아래쪽 PE(1,1)으로 전송합니다.

### 역할 요약

- **계산**: C[0][1] = A_received × B[0][1] = 2 × 2 = **4** (2×2 행렬에서 단일 항)
- **수신**: A[0][0] ← PE(0,0)
- **송신**: B[0][1]=2 → PE(1,1)

### 핀 배치

| 핀 | 상수 | 방향 | 대상 |
|----|------|------|------|
| D2 | `CLOCK_A_RX` | INPUT  | PE(0,0) CLOCK_A_TX |
| D4 | `DATA_A_RX`  | INPUT  | PE(0,0) DATA_A_TX |
| D5 | `DATA_B_TX`  | OUTPUT | PE(1,1) DATA_B_RX |
| D3 | `CLOCK_B_TX` | OUTPUT | PE(1,1) CLOCK_B_RX |

### 전체 코드

```cpp
// pe01_os.ino — PE(0,1) Output Stationary 모드
// 역할: A값 수신(왼쪽에서), B01 자신 소유, MAC 후 B값→아래쪽 전송

#define CLOCK_A_RX 2  // A값 클럭 ← PE(0,0)
#define DATA_A_RX 4   // A값 데이터 ← PE(0,0)
#define DATA_B_TX 5   // B값 데이터 → PE(1,1)
#define CLOCK_B_TX 3  // B값 클럭  → PE(1,1)

int B01 = 2;  // B[0][1]
int localSum = 0;

byte receiveByte(byte clockPin, byte dataPin) {
  byte value = 0;
  for (int i = 0; i < 8; i++) {
    value = (value << 1) | digitalRead(dataPin);
    while (digitalRead(clockPin) == HIGH);   // LOW 대기
    while (digitalRead(clockPin) == LOW);    // 다음 HIGH 대기
  }
  return value;
}

void setup() {
  pinMode(CLOCK_A_RX, INPUT);
  pinMode(DATA_A_RX, INPUT);
  pinMode(DATA_B_TX, OUTPUT);
  pinMode(CLOCK_B_TX, OUTPUT);

  // A값 수신 대기
  while (digitalRead(CLOCK_A_RX) == LOW);
  int A_received = receiveByte(CLOCK_A_RX, DATA_A_RX);

  // MAC 연산
  localSum += A_received * B01;

  // B값을 아래쪽(PE(1,1))으로 전송
  shiftOut(DATA_B_TX, CLOCK_B_TX, MSBFIRST, (byte)B01);
}

void loop() {}
```

### `receiveByte` 함수 동작

```cpp
byte receiveByte(byte clockPin, byte dataPin) {
  byte value = 0;
  for (int i = 0; i < 8; i++) {
    value = (value << 1) | digitalRead(dataPin);  // 비트 누적
    while (digitalRead(clockPin) == HIGH);         // LOW 전환 대기
    while (digitalRead(clockPin) == LOW);          // 다음 HIGH 대기
  }
  return value;
}
```

클럭 감지와 데이터 읽기를 분리하여 타이밍 오류를 방지합니다. 8비트를 MSB 순서로 수신하며, `shiftOut(MSBFIRST)`로 전송된 신호와 정확히 호환됩니다.

### 행렬 값 변경 방법

```cpp
int B01 = 5;  // B[0][1]을 5로 변경
```

## PE(1,0) 코드

PE(1,0)은 B값을 PE(0,0)으로부터 수신하고, A[1][0]=4는 자신이 소유합니다. MAC 연산 후 A값을 오른쪽 PE(1,1)으로 전송합니다.

### 역할 요약

- **계산**: C[1][0] = A[1][0] × B_received = 4 × 1 = **4** (2×2 행렬에서 단일 항)
- **수신**: B[0][0] ← PE(0,0)
- **송신**: A[1][0]=4 → PE(1,1)

### 핀 배치

| 핀 | 상수 | 방향 | 대상 |
|----|------|------|------|
| D2 | `CLOCK_B_RX` | INPUT  | PE(0,0) CLOCK_B_TX |
| D4 | `DATA_B_RX`  | INPUT  | PE(0,0) DATA_B_TX |
| D5 | `DATA_A_TX`  | OUTPUT | PE(1,1) DATA_A_RX |
| D3 | `CLOCK_A_TX` | OUTPUT | PE(1,1) CLOCK_A_RX |

### 전체 코드

```cpp
// pe10_os.ino — PE(1,0) Output Stationary 모드
// 역할: B값 수신(위쪽에서), A10 자신 소유, MAC 후 A값→오른쪽 전송

#define CLOCK_B_RX 2  // B값 클럭 ← PE(0,0)
#define DATA_B_RX 4   // B값 데이터 ← PE(0,0)
#define DATA_A_TX 5   // A값 데이터 → PE(1,1)
#define CLOCK_A_TX 3  // A값 클럭  → PE(1,1)

int A10 = 4;  // A[1][0]
int localSum = 0;

byte receiveByte(byte clockPin, byte dataPin) {
  byte value = 0;
  for (int i = 0; i < 8; i++) {
    value = (value << 1) | digitalRead(dataPin);
    while (digitalRead(clockPin) == HIGH);   // LOW 대기
    while (digitalRead(clockPin) == LOW);    // 다음 HIGH 대기
  }
  return value;
}

void setup() {
  pinMode(CLOCK_B_RX, INPUT);
  pinMode(DATA_B_RX, INPUT);
  pinMode(DATA_A_TX, OUTPUT);
  pinMode(CLOCK_A_TX, OUTPUT);

  // B값 수신 대기
  while (digitalRead(CLOCK_B_RX) == LOW);
  int B_received = receiveByte(CLOCK_B_RX, DATA_B_RX);

  // MAC 연산
  localSum += A10 * B_received;

  // A값을 오른쪽(PE(1,1))으로 전송
  shiftOut(DATA_A_TX, CLOCK_A_TX, MSBFIRST, (byte)A10);
}

void loop() {}
```

### 행렬 값 변경 방법

```cpp
int A10 = 6;  // A[1][0]을 6으로 변경
```

## PE(1,1) 코드 (LCD + 타이밍)

PE(1,1)은 격자의 끝점으로, PE(1,0)에서 A값을, PE(0,1)에서 B값을 각각 수신합니다. 두 신호가 동시에 도달할 수 있으므로 폴링 루프로 비동기 수신을 처리합니다. MAC 연산 완료 후 경과 시간과 결과를 LCD에 출력합니다.

### 역할 요약

- **계산**: C[1][1] = A_received × B_received = 4 × 2 = **8** (2×2 행렬에서 단일 항)
- **수신**: A[1][0] ← PE(1,0) / B[0][1] ← PE(0,1)
- **타이밍**: D6(TIMING_RX) HIGH → `startTime` 기록, MAC 완료 → `endTime` 기록
- **출력**: LCD에 C[1][1] 값과 경과 시간(µs) 표시

### 핀 배치

| 핀 | 상수 | 방향 | 대상 |
|----|------|------|------|
| D2 | `CLOCK_A_RX` | INPUT | PE(1,0) CLOCK_A_TX |
| D4 | `DATA_A_RX`  | INPUT | PE(1,0) DATA_A_TX |
| D3 | `CLOCK_B_RX` | INPUT | PE(0,1) CLOCK_B_TX |
| D5 | `DATA_B_RX`  | INPUT | PE(0,1) DATA_B_TX |
| D6 | `TIMING_RX`  | INPUT | PE(0,0) TIMING_PIN |

### 전체 코드

```cpp
// pe11_os.ino — PE(1,1) Output Stationary 모드
// 역할: 격자의 끝점. 두 값 수신, MAC 연산, LCD에 결과 및 시간 출력

#include <Wire.h>
#include <LiquidCrystal_I2C.h>

#define CLOCK_A_RX 2  // A값 클럭 ← PE(1,0)
#define DATA_A_RX 4   // A값 데이터 ← PE(1,0)
#define CLOCK_B_RX 3  // B값 클럭 ← PE(0,1)
#define DATA_B_RX 5   // B값 데이터 ← PE(0,1)
#define TIMING_RX 6   // 타이밍 신호 ← PE(0,0)

LiquidCrystal_I2C lcd(0x27, 16, 2);
int localSum = 0;

byte receiveByte(byte clockPin, byte dataPin) {
  byte value = 0;
  for (int i = 0; i < 8; i++) {
    value = (value << 1) | digitalRead(dataPin);
    while (digitalRead(clockPin) == HIGH);   // LOW 대기
    while (digitalRead(clockPin) == LOW);    // 다음 HIGH 대기
  }
  return value;
}

void setup() {
  pinMode(CLOCK_A_RX, INPUT);
  pinMode(DATA_A_RX, INPUT);
  pinMode(CLOCK_B_RX, INPUT);
  pinMode(DATA_B_RX, INPUT);
  pinMode(TIMING_RX, INPUT);

  lcd.init();
  lcd.backlight();
  lcd.print("Waiting...");

  // 타이밍 신호 대기 → startTime 기록
  while (digitalRead(TIMING_RX) == LOW);
  unsigned long startTime = micros();

  // 두 값을 폴링으로 동시 수신
  bool a_ready = false, b_ready = false;
  int a_val = 0, b_val = 0;

  while (!a_ready || !b_ready) {
    if (!a_ready && digitalRead(CLOCK_A_RX) == HIGH) {
      a_val = receiveByte(CLOCK_A_RX, DATA_A_RX);
      a_ready = true;
    }
    if (!b_ready && digitalRead(CLOCK_B_RX) == HIGH) {
      b_val = receiveByte(CLOCK_B_RX, DATA_B_RX);
      b_ready = true;
    }
  }

  // MAC 연산
  localSum += a_val * b_val;
  unsigned long endTime = micros();
  unsigned long elapsed = endTime - startTime;

  // LCD 출력
  lcd.clear();
  lcd.setCursor(0, 0);
  lcd.print("C11=");
  lcd.print(localSum);
  lcd.setCursor(0, 1);
  lcd.print(elapsed);
  lcd.print("us");
}

void loop() {}
```

### 타이밍 측정 원리

```
PE(0,0) MAC 직전 → D6 HIGH
                        ↓
PE(1,1) TIMING_RX HIGH 감지 → startTime = micros()
                        ↓
폴링 루프: A값, B값 수신
                        ↓
localSum += a_val * b_val → endTime = micros()
                        ↓
elapsed = endTime - startTime
```

`elapsed`는 **첫 번째 MAC(PE(0,0))이 시작된 순간**부터 **마지막 MAC(PE(1,1))이 완료된 순간**까지의 전체 병렬 처리 시간을 나타냅니다.

### 폴링 루프 — 비동기 이중 수신

```cpp
while (!a_ready || !b_ready) {
  if (!a_ready && digitalRead(CLOCK_A_RX) == HIGH) {
    a_val = receiveByte(CLOCK_A_RX, DATA_A_RX);
    a_ready = true;
  }
  if (!b_ready && digitalRead(CLOCK_B_RX) == HIGH) {
    b_val = receiveByte(CLOCK_B_RX, DATA_B_RX);
    b_ready = true;
  }
}
```

PE(1,0)과 PE(0,1)의 전송 완료 시점이 다를 수 있으므로, 두 신호를 순서에 관계없이 모두 받을 때까지 반복합니다. `a_ready`와 `b_ready` 플래그로 중복 수신을 방지합니다.

### LCD 출력 해석

업로드 및 리셋 후 LCD에 다음과 같이 표시됩니다.

```
C11=8
XXXus
```

| 표시 항목 | 의미 |
|----------|------|
| `C11=8`  | C[1][1] 연산 결과 (기본값 기준) |
| `XXXus`  | PE(0,0) MAC 시작~PE(1,1) MAC 완료까지 경과 시간(µs) |

<Tip>
LCD 주소가 `0x27`이 아닌 경우(예: `0x3F`) `LiquidCrystal_I2C lcd(0x3F, 16, 2);`로 변경하세요. I2C 주소는 I2C 스캐너 스케치로 확인할 수 있습니다.
</Tip>

## 업로드 순서

4개의 보드를 올바른 순서로 업로드하지 않으면 PE(0,0)이 신호를 보낼 때 수신 보드가 아직 준비되지 않아 데이터가 유실됩니다.

<Steps>
  ### PE(0,1) 업로드
  `pe01_os.ino`를 PE(0,1) 보드에 업로드합니다. 업로드 완료 후 해당 보드는 `CLOCK_A_RX`(D2) 핀이 HIGH가 될 때까지 대기 상태에 들어갑니다.

  ### PE(1,0) 업로드
  `pe10_os.ino`를 PE(1,0) 보드에 업로드합니다. 업로드 완료 후 해당 보드는 `CLOCK_B_RX`(D2) 핀이 HIGH가 될 때까지 대기 상태에 들어갑니다.

  ### PE(1,1) 업로드
  `pe11_os.ino`를 PE(1,1) 보드에 업로드합니다. LCD에 `Waiting...`이 표시되면 정상적으로 대기 중인 것입니다. `TIMING_RX`(D6) 신호를 기다립니다.

  ### PE(0,0) 업로드 (마지막)
  `pe00_os.ino`를 PE(0,0) 보드에 업로드합니다. 업로드 직후 보드가 자동 리셋되며 `delay(500)` 후 타이밍 신호를 보내고 전체 연산이 시작됩니다. 잠시 후 PE(1,1)의 LCD에 결과가 표시됩니다.
</Steps>

<Warning>
4개 보드를 모두 재실행하려면 PE(0,1) → PE(1,0) → PE(1,1) 순서로 Reset을 누른 뒤, 마지막으로 PE(0,0)의 Reset을 누르세요. PE(0,0)을 먼저 리셋하면 나머지 보드가 준비되기 전에 신호가 전송됩니다.
</Warning>

## 전체 결선 요약

아래 표는 보드 간 물리적 연결을 정리한 것입니다. 같은 행의 두 핀을 점퍼 와이어로 연결합니다.

| 송신 보드 | 송신 핀 | 수신 보드 | 수신 핀 | 신호 |
|----------|---------|----------|---------|------|
| PE(0,0) | D4 (DATA_A_TX)  | PE(0,1) | D4 (DATA_A_RX)  | A[0][0] 데이터 |
| PE(0,0) | D2 (CLOCK_A_TX) | PE(0,1) | D2 (CLOCK_A_RX) | A[0][0] 클럭   |
| PE(0,0) | D5 (DATA_B_TX)  | PE(1,0) | D4 (DATA_B_RX)  | B[0][0] 데이터 |
| PE(0,0) | D3 (CLOCK_B_TX) | PE(1,0) | D2 (CLOCK_B_RX) | B[0][0] 클럭   |
| PE(0,0) | D6 (TIMING_PIN) | PE(1,1) | D6 (TIMING_RX)  | 타이밍 신호     |
| PE(0,1) | D5 (DATA_B_TX)  | PE(1,1) | D5 (DATA_B_RX)  | B[0][1] 데이터 |
| PE(0,1) | D3 (CLOCK_B_TX) | PE(1,1) | D3 (CLOCK_B_RX) | B[0][1] 클럭   |
| PE(1,0) | D5 (DATA_A_TX)  | PE(1,1) | D4 (DATA_A_RX)  | A[1][0] 데이터 |
| PE(1,0) | D3 (CLOCK_A_TX) | PE(1,1) | D2 (CLOCK_A_RX) | A[1][0] 클럭   |

모든 보드의 GND 핀은 공통 GND에 연결해야 통신이 안정적으로 동작합니다.
