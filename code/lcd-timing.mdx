SAVANT 실험에서는 4개의 Arduino Nano 보드가 USB 허브에 동시에 연결되어 있기 때문에, 시리얼 모니터로 각 보드의 출력을 개별 확인하는 것이 현실적으로 번거롭습니다. 이를 해결하기 위해 최종 결과를 출력하는 PE(1,1) 보드에 I2C LCD 16×2 모듈을 연결합니다. LCD에는 최종 계산 결과인 C[1][1]과 처리 소요 시간(μs)이 표시되어, 보드를 노트북에서 분리한 상태에서도 실험 결과를 즉시 확인할 수 있습니다.

## I2C LCD 16×2 배선

I2C 방식 LCD는 SDA·SCL 두 선만으로 Arduino와 통신하므로 배선이 간단합니다. PE(1,1) 보드에만 연결합니다.

| LCD 핀 | Arduino Nano 핀 | 설명 |
|--------|----------------|------|
| VCC | 5V | 전원 |
| GND | GND | 접지 |
| SDA | A4 | I2C 데이터 |
| SCL | A5 | I2C 클럭 |

<Note>
Arduino Nano의 I2C 핀은 A4(SDA)와 A5(SCL)로 고정되어 있습니다. Wire 라이브러리는 이 핀을 자동으로 사용하므로 별도 핀 설정 없이 `lcd.init()`만 호출하면 됩니다.
</Note>

### I2C 주소 확인 방법

대부분의 I2C LCD 백팩 모듈은 기본 주소 `0x27`을 사용하지만, 일부 제조사 모듈은 `0x3F`를 사용합니다. 주소가 맞지 않으면 LCD 화면이 켜지지 않습니다.

```cpp
// I2C 주소 스캔 코드 — 업로드 후 시리얼 모니터에서 확인
#include <Wire.h>

void setup() {
  Wire.begin();
  Serial.begin(9600);
  Serial.println("I2C 주소 스캔 중...");

  for (byte addr = 1; addr < 127; addr++) {
    Wire.beginTransmission(addr);
    byte error = Wire.endTransmission();
    if (error == 0) {
      Serial.print("장치 발견: 0x");
      Serial.println(addr, HEX);
    }
  }
  Serial.println("스캔 완료.");
}

void loop() {}
```

출력 예시:
```
장치 발견: 0x27
스캔 완료.
```

확인된 주소를 코드의 `LiquidCrystal_I2C lcd(0x27, 16, 2);` 부분에 입력합니다.

## D6 타이밍 신호 원리

SAVANT의 처리 시간 측정은 **첫 번째 MAC 연산 직전**부터 **마지막 MAC 연산 직후**까지를 기준으로 합니다. 이 방식을 채택한 이유와 구현 방법을 아래에 설명합니다.

### 왜 MAC 직전을 기준으로 하는가

단일 보드 측정에서도 첫 번째 곱셈·누산 연산이 시작되는 시점을 기준으로 처리 시간을 측정합니다. 격자 구조에서 PE(0,0)이 **데이터를 전송하기 전**이 아니라 **자신의 MAC 연산 직전**에 D6를 HIGH로 올리는 이유는, 단일 보드와 동일한 기준점을 사용하기 위해서입니다. 만약 전송 이전을 기준으로 삼으면 통신 준비 오버헤드가 측정값에 포함되어 두 조건 간의 비교가 공정하지 않게 됩니다.

| 기준점 | 단일 보드 | 2×2 격자 |
|--------|---------|---------|
| 측정 시작 | 첫 번째 MAC 직전 | D6 HIGH (PE(0,0)의 첫 MAC 직전) |
| 측정 종료 | 마지막 MAC 직후 | D6 수신 후 PE(1,1)의 마지막 MAC 직후 |
| 포함 범위 | 순수 연산 시간 | 연산 + 보드 간 통신 시간 |

<Note>
격자 구조의 측정값에는 PE 간 시리얼 통신 지연이 포함됩니다. 이 차이가 실험의 핵심 관찰 대상이며, 단일 보드 대비 격자의 오버헤드를 정량화하는 근거가 됩니다.
</Note>

### PE(0,0) D6 출력 코드

```cpp
// PE(0,0) — D6 타이밍 신호 출력
#define PIN_D6 6

// setup() 내
pinMode(PIN_D6, OUTPUT);
digitalWrite(PIN_D6, LOW);

// loop() 내 — 첫 번째 MAC 직전
digitalWrite(PIN_D6, HIGH);   // ← 타이밍 시작
int mac_result = weight * input;  // 첫 번째 MAC 연산
```

### PE(1,1) D6 모니터링 코드

PE(1,1)은 D6 핀을 입력으로 설정하고, HIGH 신호가 감지되는 순간 `micros()`로 시작 시각을 기록합니다.

```cpp
// PE(1,1) — D6 타이밍 신호 감지 및 시간 측정
#define PIN_D6 6

unsigned long startTime = 0;
unsigned long endTime   = 0;

// setup() 내
pinMode(PIN_D6, INPUT);

// loop() 내 — 측정 시작
while (digitalRead(PIN_D6) == LOW) {
  // D6 HIGH 대기 (PE(0,0)의 첫 MAC 시작 신호)
}
startTime = micros();

// --- 모든 수신 및 MAC 연산 수행 ---
// ... (부분합 수신 및 자체 MAC 코드) ...

// 마지막 MAC 완료 직후
endTime = micros();
unsigned long elapsed = endTime - startTime;
```

## LCD 출력 형식

PE(1,1)의 LCD는 두 줄로 구성됩니다.

```
[1번째 줄]  C[1][1]=28
[2번째 줄]  Time=1423us
```

| 줄 | 내용 | 예시 |
|----|------|------|
| 1줄 | 최종 계산 결과 C[1][1] | `C[1][1]=28` |
| 2줄 | 경과 시간 (마이크로초) | `Time=1423us` |

### LCD 출력 코드

```cpp
#include <Wire.h>
#include <LiquidCrystal_I2C.h>

LiquidCrystal_I2C lcd(0x27, 16, 2);

// setup() 내
lcd.init();
lcd.backlight();

// loop() 내 — 측정 완료 후
lcd.clear();

// 1번째 줄: 결과값
lcd.setCursor(0, 0);
lcd.print("C[1][1]=");
lcd.print(finalResult);   // int형 결과

// 2번째 줄: 경과 시간
lcd.setCursor(0, 1);
lcd.print("Time=");
lcd.print(elapsed);       // unsigned long, 마이크로초
lcd.print("us");
```

<Note>
16×2 LCD의 한 줄에는 최대 16자가 표시됩니다. 결과값이나 시간이 길어지면 잘릴 수 있습니다. 결과값이 4자리 이상이거나 시간이 99999μs를 초과하면 `lcd.print(elapsed/1000); lcd.print("ms");` 형식으로 밀리초 단위로 전환하십시오.
</Note>

## 결과 검증 방법

LCD에 표시된 C[1][1] 값이 이론값과 일치하는지 확인합니다.

### 기본값(2×2×2) 기대 결과

A = [[2,3],[4,5]], B = [[1,2],[3,4]] 일 때:

| 행렬 원소 | 계산식 | 기댓값 |
|----------|--------|-------|
| C[0][0] | 2×1 + 3×3 | 11 |
| C[0][1] | 2×2 + 3×4 | 16 |
| C[1][0] | 4×1 + 5×3 | 19 |
| C[1][1] | 4×2 + 5×4 | **28** |

LCD 1번째 줄에 `C[1][1]=28`이 표시되면 계산이 정상입니다.

<Tip>
C[0][p] 값(PE(0,1)이 계산)은 LCD에 표시되지 않습니다. 이 값을 검증하려면 실험 전 PE(0,1) 보드를 노트북에 단독 연결하고 시리얼 모니터(9600 baud)를 열어 출력 로그를 확인합니다. 정상이면 `[PE(0,1)] partialSum=9` (W[0][1]×B[0][0] = 3×3 = 9) 등의 로그가 출력됩니다.
</Tip>

### 검증 절차

<Steps>
  <Step title="이론값 사전 계산">
    실험 전 손으로 또는 Python으로 A×B를 계산하여 C[1][1] 기댓값을 기록합니다.
  </Step>
  <Step title="LCD 값과 대조">
    측정 완료 후 LCD 1번째 줄의 숫자와 기댓값을 비교합니다. 일치하면 정상, 불일치하면 코드 또는 배선을 점검합니다.
  </Step>
  <Step title="불일치 시 디버깅">
    PE(1,1)을 노트북에 연결하고 시리얼 모니터를 열어 중간 수신값(`ps_up`, `ps_left`, `myMAC`)을 확인합니다. 어느 단계에서 값이 틀렸는지 추적합니다.
  </Step>
</Steps>

<Warning>
D6 배선 시 PE(0,0)의 D6 핀(출력)과 PE(1,1)의 D6 핀(입력)을 직접 점퍼 와이어로 연결합니다. 두 보드가 USB 허브를 통해 공통 GND를 공유하고 있는지 반드시 확인하십시오. GND가 연결되지 않으면 HIGH/LOW 신호가 정상적으로 전달되지 않습니다.
</Warning>
