SAVANT 실험은 Arduino Nano 기반의 단일 보드 순차 처리와 2×2 시스톨릭 어레이 병렬 처리를 공정하게 비교하기 위해 설계되었습니다. 연구의 신뢰성을 확보하기 위해 모든 조건을 사전에 명시하고, 결과에 유리한 조건만 선별하지 않으며, 예상치 못한 결과(예: 격자가 단일 보드보다 느린 경우)도 그대로 보고하는 원칙을 따릅니다. 이 페이지에서는 연구 질문, 변인 구성, 전체 측정 조건 표, 측정 프로토콜, 연구 무결성 원칙을 설명합니다.

## 연구 질문

SAVANT 프로젝트는 다음 세 가지 연구 질문에 답하는 것을 목표로 합니다.

1. **처리 구조의 효과**: 동일한 행렬 크기에서 2×2 격자 구조는 단일 보드 순차 처리보다 처리 시간이 짧은가? 격자의 이점이 나타나는 최소 행렬 크기(교차점)는 어디인가?

2. **데이터 흐름 방식의 효과**: 동일한 격자 구조에서 OS(Output Stationary) 모드와 WS(Weight Stationary) 모드 중 어느 쪽이 처리 시간 및 통신 접근 횟수 면에서 효율적인가?

3. **이론값과 실측값의 일치**: 이론 클럭 수 공식(Nc = 2×S_R + S_C + T − 2)이 실제 측정 클럭 수와 얼마나 일치하는가? 불일치가 발생한다면 그 원인은 무엇인가?

## 변인 구성

### 독립변인

| # | 독립변인 | 수준 | 비고 |
|---|---------|------|------|
| 1 | 처리 구조 | 단일 보드 / 2×2 격자 | 하드웨어 구성 |
| 2 | 데이터 흐름 방식 | OS / WS | 격자 조건에만 해당 |
| 3 | 행렬 크기(M×N×P) | 2×2×2 / 2×2×4 / 2×4×2 | A는 M×N, B는 N×P |

**행렬 크기 표기 규칙**: M×N×P에서 A는 M행 N열, B는 N행 P열, 결과 C는 M행 P열입니다.

### 종속변인

| # | 종속변인 | 측정 단위 | 측정 방법 |
|---|---------|---------|---------|
| 1 | 처리 시간 | μs (마이크로초) | D6 신호 + `micros()` |
| 2 | 메모리·통신 접근 횟수 | 횟수 (count) | 코드 내 카운터 변수 집계 |

<Note>
처리 시간은 첫 번째 MAC 연산 시작부터 마지막 MAC 연산 완료까지를 기준으로 합니다. 단일 보드에서는 `micros()` 직접 호출, 격자에서는 D6 타이밍 신호를 통해 동일한 기준점을 유지합니다.
</Note>

## 9가지 측정 조건

총 9가지 조건은 독립변인 3개의 조합으로 구성됩니다. 단일 보드는 데이터 흐름 방식이 없으므로 3가지 조건, 격자는 OS/WS × 3가지 행렬 크기 = 6가지 조건입니다.

| 조건 # | 처리 방식 | 데이터 흐름 | 행렬 크기 (M×N×P) |
|--------|----------|-----------|-----------------|
| 1 | 단일 보드 | — | 2×2×2 |
| 2 | 단일 보드 | — | 2×2×4 |
| 3 | 단일 보드 | — | 2×4×2 |
| 4 | 2×2 격자 | OS | 2×2×2 |
| 5 | 2×2 격자 | OS | 2×2×4 |
| 6 | 2×2 격자 | OS | 2×4×2 |
| 7 | 2×2 격자 | WS | 2×2×2 |
| 8 | 2×2 격자 | WS | 2×2×4 |
| 9 | 2×2 격자 | WS | 2×4×2 |

<Note>
조건 5와 6에서 OS 모드의 S_C = P이므로, 행렬 크기가 커지면 S_C가 격자 열 수(2)를 초과할 수 있습니다. 이 경우 타일링(tiling)을 적용하여 2라운드로 나누어 처리합니다. 타일링 여부도 기록 항목에 포함됩니다.
</Note>

## 측정 프로토콜

### 반복 횟수 및 통계 처리

- 각 조건당 최소 **5회** 반복 측정
- 측정값에서 **평균(mean)**과 **표준편차(SD)** 계산
- 이상값(outlier) 판정: 평균 ± 2×SD 범위를 벗어난 값은 별도 표시 후 원인 기록(배선 접촉 불량, 측정 오류 등)
- 이상값이 있을 경우 해당 회차를 제외한 나머지로 재계산하되, 원본 데이터는 삭제하지 않고 유지

### 리셋 버튼을 이용한 반복 측정

코드 재업로드 없이 Arduino의 리셋(RESET) 버튼을 눌러 측정을 반복합니다.

<Tip>
리셋 버튼으로 재실행하면 모든 변수가 초기화되고 `setup()`부터 다시 실행됩니다. 각 측정 후 LCD 값을 기록하고 리셋을 누르면 추가 업로드 없이 연속으로 5회 측정이 가능합니다. 단, PE(0,0)을 마지막에 리셋해야 다른 PE들이 준비 상태가 된 후 신호가 전송됩니다.
</Tip>

### 스프레드시트 기록 항목

각 측정 회차마다 다음 항목을 기록합니다.

| 열 | 항목 | 비고 |
|----|------|------|
| A | 조건 번호 | 1~9 |
| B | 처리 방식 | 단일/격자 |
| C | 데이터 흐름 | OS/WS/— |
| D | 행렬 크기 | 예: 2×2×2 |
| E~I | 회차 1~5 처리 시간 (μs) | |
| J | 평균 처리 시간 (μs) | |
| K | 표준편차 (μs) | |
| L~P | 회차 1~5 접근 횟수 | |
| Q | 평균 접근 횟수 | |
| R | 이상값 여부 | Y/N |
| S | 비고 | 특이사항 기록 |

## 연구 무결성 원칙

<Warning>
아래 원칙들은 실험 결과의 신뢰성과 연구 윤리를 위해 반드시 준수해야 합니다. 이 원칙을 위반하면 연구 결과 전체의 타당성이 훼손됩니다.
</Warning>

### 원칙 1: 9가지 조건 전체 보고

실험에서 설정한 9가지 조건은 결과에 관계없이 모두 보고합니다. 격자가 단일 보드보다 느리게 나온 조건이 있어도 해당 조건을 제외하거나 은폐하지 않습니다. 예상과 다른 결과는 오히려 통신 오버헤드, 격자 구조의 한계 등을 분석하는 근거가 됩니다.

### 원칙 2: 인위적 지연 추가 금지

단일 보드 코드에 `delay()`, `delayMicroseconds()` 등의 인위적 지연을 추가하여 처리 시간을 부풀리는 행위는 허용되지 않습니다. 두 조건은 순수 연산과 통신에 소요되는 시간만을 비교해야 합니다.

### 원칙 3: 통신 최적화는 독립변인으로 처리

OS 모드와 WS 모드의 차이(전송 데이터 종류)는 사후 보정(post-hoc fix)이 아니라 사전에 설계된 독립변인입니다. "격자가 느리게 나왔으니 WS 모드로 바꾼다"는 방식이 아니라, OS와 WS를 별도 조건으로 독립적으로 측정하고 비교합니다.

### 원칙 4: 격자가 단일 보드보다 느린 결과의 해석

2×2 격자가 단일 보드보다 처리 시간이 길게 측정되면, 이것은 **유효하고 해석 가능한 결과**입니다. 소규모 행렬에서는 PE 간 시리얼 통신 오버헤드가 병렬 처리 이득을 상회할 수 있으며, 이를 정량화하는 것 자체가 연구의 기여입니다. 행렬 크기가 커질수록 격자가 유리해지는 교차점(crossover point)을 찾는 것도 연구 목표 중 하나입니다.
