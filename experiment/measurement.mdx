SAVANT 실험을 실제로 수행할 때는 코드 업로드 순서, LCD 읽기 방법, 리셋 버튼 반복 측정, 행렬 크기 변경 방법을 체계적으로 따라야 일관성 있는 데이터를 얻을 수 있습니다. 이 페이지는 조건 1(단일 보드, 2×2×2)부터 조건 9(2×2 격자, WS, 2×4×2)까지 9가지 조건을 순서대로 측정하기 위한 단계별 절차를 안내합니다.

## 사전 준비 체크리스트

측정을 시작하기 전에 아래 항목을 모두 확인합니다.

| 항목 | 확인 방법 |
|------|---------|
| 9가지 조건 코드 파일 준비 완료 | 각 조건의 `.ino` 파일이 PC에 저장되어 있는지 확인 |
| 스프레드시트 템플릿 열기 | 행: 조건 1~9 / 열: 회차 1~5, 평균, SD |
| USB 허브 및 케이블 점검 | 4포트 허브에 보드 4개 연결, 전원 공급 확인 |
| D6 점퍼 와이어 연결 | PE(0,0) D6 핀 → PE(1,1) D6 핀, GND 공통 연결 |
| I2C LCD 배선 확인 | PE(1,1)의 A4(SDA), A5(SCL), 5V, GND 연결 |
| LCD 주소 확인 | I2C 스캔 코드로 `0x27` 또는 `0x3F` 확인 |
| 이론 기댓값 계산 완료 | 각 조건의 C[1][1] 기댓값을 손계산 또는 Python으로 사전 계산 |

<Note>
스프레드시트의 각 셀에는 측정값을 바로 입력할 수 있도록 조건 번호, 처리 방식, 데이터 흐름, 행렬 크기 열을 미리 채워두면 실험 중 실수를 줄일 수 있습니다.
</Note>

## 단일 보드 측정 절차 (조건 1~3)

단일 보드 조건에서는 Arduino Nano 1개만 사용하며, 노트북에 직접 연결하여 시리얼 모니터로 결과를 읽습니다.

<Steps>
  <Step title="코드 준비 및 행렬 크기 설정">
    단일 보드용 `.ino` 파일을 열고 `#define` 값을 해당 조건에 맞게 설정합니다.

    ```cpp
    // 조건 1: 2×2×2
    #define M_SIZE 2
    #define N_SIZE 2
    #define P_SIZE 2

    // 조건 2: 2×2×4
    #define M_SIZE 2
    #define N_SIZE 2
    #define P_SIZE 4

    // 조건 3: 2×4×2
    #define M_SIZE 2
    #define N_SIZE 4
    #define P_SIZE 2
    ```

    행렬 값 배열(`int A[][]`, `int B[][]`)도 크기에 맞게 수정합니다.
  </Step>
  <Step title="보드 업로드">
    Arduino Nano를 노트북 USB에 직접 연결합니다. Arduino IDE에서 보드(Arduino Nano)와 포트를 선택한 뒤 업로드합니다.
  </Step>
  <Step title="시리얼 모니터 열기">
    업로드 완료 후 Arduino IDE의 시리얼 모니터(Ctrl+Shift+M)를 엽니다. 전송 속도: 9600 baud. 보드가 자동 리셋되면서 측정이 시작됩니다.
  </Step>
  <Step title="결과 기록">
    시리얼 모니터에서 처리 시간(μs)과 접근 횟수를 확인하고 스프레드시트에 입력합니다.

    출력 예시:
    ```
    [단일보드] C[1][1]=28
    [단일보드] Elapsed=312 us
    [단일보드] Access count=8
    ```
  </Step>
  <Step title="리셋으로 5회 반복">
    보드의 RESET 버튼을 눌러 재실행합니다. 시리얼 모니터가 열려 있는 상태에서 리셋하면 새 측정값이 출력됩니다. 총 5회 반복하여 기록합니다.

    <Tip>
      RESET 버튼을 누른 후 약 2초 대기하면 `setup()`의 초기화가 완료됩니다. 초기화 완료 메시지(`[단일보드] ready.`)가 출력된 후 측정값이 나타납니다.
    </Tip>
  </Step>
  <Step title="다음 조건으로 이동">
    `#define` 값과 행렬 배열을 다음 조건에 맞게 수정하고 재업로드합니다. 조건 2, 3도 동일 절차를 반복합니다.
  </Step>
</Steps>

## 격자 측정 절차 (조건 4~9)

격자 측정에서는 4개 보드를 USB 허브에 연결하고 PE(1,1)의 LCD에서 결과를 읽습니다.

### 업로드 순서 및 방법

<Steps>
  <Step title="측정 조건 코드 선택">
    해당 조건(OS/WS, 행렬 크기)에 맞는 코드 세트를 선택합니다. 4개 보드 각각에 대한 파일이 필요합니다.

    | 조건 | 코드 세트 |
    |------|---------|
    | 조건 4~6 (OS) | `os_pe00.ino`, `os_pe01.ino`, `os_pe10.ino`, `os_pe11.ino` |
    | 조건 7~9 (WS) | `ws_pe00.ino`, `ws_pe01.ino`, `ws_pe10.ino`, `ws_pe11.ino` |
  </Step>
  <Step title="행렬 크기 #define 수정">
    4개 파일 모두에서 `#define M_SIZE`, `N_SIZE`, `P_SIZE` 값을 해당 조건에 맞게 동일하게 수정합니다.

    <Warning>
      4개 보드의 `#define` 값이 하나라도 다르면 데이터 수신 타이밍이 맞지 않아 잘못된 결과가 나옵니다. 업로드 전 4개 파일을 모두 열어 `#define` 값을 교차 확인하십시오.
    </Warning>
  </Step>
  <Step title="역순으로 업로드 (PE(1,1) 먼저)">
    PE(1,1) → PE(1,0) → PE(0,1) → PE(0,0) 순서로 각 보드를 노트북에 직접 연결하여 업로드합니다. 마스터인 PE(0,0)을 마지막에 업로드하여 모든 슬레이브 보드가 먼저 준비 상태가 되도록 합니다.
  </Step>
  <Step title="USB 허브에 연결">
    4개 보드를 모두 USB 허브에 꽂습니다. D6 점퍼 와이어와 SoftwareSerial 연결 배선이 올바른지 다시 확인합니다.
  </Step>
  <Step title="전원 인가 및 측정">
    허브에 전원을 공급합니다. PE(0,0)이 2초 대기 후 자동으로 신호 전송을 시작합니다. PE(1,1)의 LCD에 결과가 표시됩니다.

    ```
    LCD 1번째 줄: C[1][1]=28
    LCD 2번째 줄: Time=1423us
    ```
  </Step>
  <Step title="결과 기록">
    LCD에 표시된 C[1][1] 값과 Time 값을 스프레드시트에 입력합니다. C[1][1]이 이론 기댓값과 일치하는지 즉시 확인합니다. 불일치 시 해당 측정을 무효 처리하고 배선을 점검합니다.
  </Step>
  <Step title="리셋으로 5회 반복">
    PE(1,1)의 RESET 버튼을 먼저 누른 뒤, PE(0,1), PE(1,0), 마지막으로 PE(0,0) 순서로 리셋합니다. LCD가 `Waiting D6...`으로 돌아오면 준비 완료입니다.

    <Tip>
      실험 중 보드를 PC와 연결하지 않아도 USB 허브로 전원이 공급되므로 리셋 버튼만으로 반복 측정이 가능합니다. 시리얼 케이블을 뽑고 나서도 LCD로 결과를 확인할 수 있는 것이 격자 측정의 장점입니다.
    </Tip>
  </Step>
</Steps>

## 이상값 처리 방법

| 상황 | 조치 |
|------|------|
| LCD 값이 기댓값과 다름 | 해당 회차 무효 처리, 배선 확인 후 재측정 |
| 측정값이 평균 ± 2×SD 초과 | 별도 표시 후 원인 기록, 원본 보존 |
| LCD에 값이 표시되지 않음 | I2C 주소 확인, 배선 재점검 |
| 특정 회차 시간이 0으로 출력 | D6 타이밍 신호 배선 확인 |

이상값이 5회 중 1회 발생한 경우: 나머지 4회로 평균·SD를 계산하되, 원본 데이터와 이상값 발생 이유를 비고란에 기록합니다. 이상값이 2회 이상 발생하면 해당 조건을 재측정합니다.

## 조건 간 전환 요약표

| 조건 전환 | 필요 작업 |
|---------|---------|
| 같은 모드, 다른 행렬 크기 | `#define` 수정 + 배열 값 수정 + 4개 보드 재업로드 |
| 단일→격자 또는 격자→단일 | 다른 코드 세트 선택 + 배선 변경 (격자는 배선 추가) |
| OS → WS (같은 격자) | WS 코드 세트로 재업로드 (배선 동일) |
| 반복 측정 (같은 조건) | RESET 버튼만 사용 (재업로드 불필요) |

<Note>
조건 순서는 1→9로 진행하되, 단일 보드 조건(1~3)을 먼저 완료한 뒤 격자 조건(4~9)으로 넘어가면 배선 변경 횟수를 최소화할 수 있습니다. 격자 조건 내에서는 OS(4~6) → WS(7~9) 순서를 권장합니다.
</Note>
