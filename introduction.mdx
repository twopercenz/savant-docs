SAVANT(**S**ystolic **A**rray as **V**on Neumann's **A**nswer to the bottleneck **T**hreshold)는 고등학교 수준의 탐구 프로젝트로, 아두이노 나노 5대를 활용해 **폰 노이만 순차 처리 구조**와 **2×2 시스톨릭 어레이 병렬 처리 구조**의 행렬 연산 효율을 직접 비교합니다. 인공지능 연산이 폭발적으로 증가하는 시대에, 왜 기존 컴퓨터 구조는 이 수요를 따라가지 못하는지, 그리고 하드웨어 수준의 병렬 처리가 어떤 가능성을 열어주는지를 손에 잡히는 실험으로 탐구합니다.

## SAVANT란 무엇인가?

현대 AI 모델은 수백만~수십억 개의 행렬 곱셈을 반복 수행합니다. 이 연산을 처리하는 전통적인 방식—폰 노이만 구조—은 CPU와 메모리 사이에서 데이터를 끊임없이 주고받아야 하는 구조적 한계, 즉 **메모리 병목(Memory Bottleneck)**을 가지고 있습니다. 시스톨릭 어레이는 이 한계를 극복하기 위해 설계된 병렬 하드웨어 구조로, Google TPU 같은 현대 AI 가속기의 핵심 원리이기도 합니다.

SAVANT는 이 두 구조를 아두이노라는 저비용 하드웨어로 재현하여, 처리 시간·클록 수·메모리 접근 횟수를 직접 측정하고 비교합니다.

## 이 프로젝트가 중요한 이유

AI의 에너지 소비 문제는 단순한 소프트웨어 최적화로 해결하기 어렵습니다. ChatGPT 한 번의 응답이 일반 웹 검색보다 약 7배 많은 에너지를 소비한다는 연구 결과(de Vries, 2023)가 있습니다. 이 비효율의 구조적 원인 중 하나가 바로 폰 노이만 병목입니다. 시스톨릭 어레이는 데이터를 PE(처리 요소) 간에 직접 전달하여 메모리 접근 횟수 자체를 줄이는 방식으로 이 문제에 접근합니다.

물론 폰 노이만 구조가 AI 에너지 문제의 유일한 원인은 아닙니다. SAVANT는 이 구조적 비효율을 **하나의 기여 요인**으로 엄밀하게 탐구합니다.

<Note>
  아두이노 나노(ATmega328)는 엄밀히 말해 수정된 하버드 아키텍처입니다. SAVANT의 제어군은 하버드 아키텍처 위에서 **폰 노이만식 메모리 접근 패턴**(공유 배열 순차 읽기/쓰기)을 소프트웨어적으로 모사합니다. 이 점은 실험의 중요한 한계이자 논의 포인트입니다.
</Note>

## 이 문서는 누구를 위한 것인가?

이 문서는 컴퓨터 구조나 병렬 처리를 처음 접하는 **고등학생 및 입문자**를 주요 독자로 합니다. 아두이노 기초 코딩 경험이 있다면 충분히 따라올 수 있도록 설계되었으며, 전문 용어는 처음 등장할 때마다 쉬운 말로 설명합니다. 이미 배경지식이 있는 독자라면 각 페이지의 기술 세부 사항 섹션을 중심으로 읽어도 됩니다.

## 이 문서를 읽는 순서

이 문서는 개념 이해 → 실험 설계 → 구현 → 결과 분석 순서로 구성되어 있습니다. 처음 접하는 분이라면 아래 순서를 따르는 것을 권장합니다.

<Steps>
  <Step title="소개 (현재 페이지)">
    SAVANT가 무엇인지, 왜 만들었는지 큰 그림을 이해합니다.
  </Step>
  <Step title="실험 설계">
    독립변인(처리 구조, 데이터 흐름 모드, 행렬 크기), 종속변인(처리 시간, 클록 수, 메모리 접근 횟수), 9가지 측정 조합을 확인합니다.
  </Step>
  <Step title="하드웨어 구성">
    아두이노 나노 5대와 브레드보드, I2C LCD, 점퍼선으로 실험 장치를 조립하는 방법을 배웁니다.
  </Step>
  <Step title="소프트웨어 구현">
    제어군의 순차 연산 코드, 실험군의 OS/WS 데이터 흐름 코드, PE(1,1) 동시 수신 문제 해결 방법 등을 다룹니다.
  </Step>
  <Step title="결과 및 분석">
    45회(9조건 × 5회) 측정 결과, 이론 클록 수와 실측값 비교, 그리고 UART 통신 오버헤드가 결과에 미치는 영향을 분석합니다.
  </Step>
</Steps>

## 핵심 용어 미리 보기

이 문서에 자주 등장하는 용어를 미리 정리합니다. 각 용어의 자세한 설명은 해당 페이지에서 다룹니다.

| 용어 | 간단 설명 |
|------|-----------|
| 폰 노이만 병목 | CPU와 메모리 사이의 단일 버스가 데이터 전송 속도를 제한하는 현상 |
| 시스톨릭 어레이 | 처리 요소(PE)들이 격자를 이루어 데이터를 직접 전달하며 병렬 연산하는 구조 |
| PE (Processing Element) | 시스톨릭 어레이의 각 연산 노드. SAVANT에서는 아두이노 나노 1대가 1개의 PE 역할 |
| OS (Output Stationary) | 각 PE가 자신의 출력값(누산 결과)을 고정하고 입력 데이터를 흘려보내는 방식 |
| WS (Weight Stationary) | 각 PE가 가중치(행렬 A의 원소)를 고정하고 입력 벡터를 흘려보내는 방식 |
| MAC | Multiply-Accumulate. 곱하고 더하는 연산으로, 행렬 곱셈의 기본 단위 |
| Nc | 이론적 클록 수. 공식: Nc = 2×S_R + S_C + T − 2 |

<Note>
  이 프로젝트의 총 예산은 약 66,300원(100,000원 이내)이며, 아두이노 나노(SZH-EK025, CH340 칩), 브레드보드, 점퍼선, I2C LCD 16×2, USB 허브로 구성됩니다. 고가의 장비 없이도 컴퓨터 구조의 핵심 원리를 탐구할 수 있다는 점이 이 프로젝트의 강점입니다.
</Note>
