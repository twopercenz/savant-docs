SAVANT 실험은 단순한 속도 비교가 아닙니다. 처리 구조(순차 vs 병렬), 데이터 흐름 방식(OS vs WS), 행렬 크기라는 세 가지 독립변인을 체계적으로 조합하여, 폰 노이만 병목이 실제 연산 효율에 어떤 방식으로 영향을 미치는지를 측정 가능한 수치로 검증합니다. 이 페이지에서는 탐구 질문의 배경, 변인 구조, 이론 모델, 측정 계획, 그리고 연구 정직성 원칙을 다룹니다.

## 탐구 질문과 동기

### AI 에너지 문제에서 출발하다

인공지능 모델의 학습과 추론에는 막대한 에너지가 소요됩니다. 연구에 따르면 대형 언어 모델의 응답 한 건은 일반 웹 검색 대비 약 7배의 에너지를 소비합니다(de Vries, 2023). 이 에너지 비효율의 구조적 원인을 추적하면, 현대 컴퓨터 대부분이 채택하고 있는 **폰 노이만 구조**에 도달합니다.

폰 노이만 구조는 CPU와 메모리가 단일 버스(통로)로 연결된 설계입니다. 연산 장치는 데이터를 처리할 때마다 메모리에서 값을 읽어오고 결과를 다시 저장해야 합니다. 행렬 곱셈처럼 동일한 데이터를 반복 참조하는 연산에서는 이 왕복이 병목이 됩니다. 이를 **폰 노이만 병목(Von Neumann Bottleneck)**이라고 합니다.

<Note>
  폰 노이만 병목은 AI 에너지 문제의 기여 요인 중 하나입니다. "폰 노이만 구조가 모든 비효율의 근본 원인"이라는 주장은 논리적 과단순화입니다. SAVANT는 이 구조적 비효율을 **하나의 독립적인 요인**으로 엄밀하게 탐구하며, 과장된 주장을 피합니다.
</Note>

### 시스톨릭 어레이: 구조로 병목을 우회하다

시스톨릭 어레이는 처리 요소(PE)들이 격자 형태로 배열되어, 인접한 PE끼리 데이터를 직접 전달하며 연산하는 구조입니다. 메모리를 반복 참조하는 대신 데이터가 PE들 사이를 흘러가면서 연산이 이루어지므로, 메모리 접근 횟수 자체가 줄어듭니다. Google의 TPU(Tensor Processing Unit)가 이 원리를 대규모로 구현한 대표적 사례입니다.

SAVANT의 핵심 탐구 질문은 다음과 같습니다.

> **"2×2 시스톨릭 어레이 구조(아두이노 4대)는 순차 처리 구조(아두이노 1대)에 비해 행렬 곱셈의 처리 효율(처리 시간, 클록 수, 메모리 접근 횟수)이 더 높은가? 데이터 흐름 방식(OS/WS)과 행렬 크기에 따라 그 차이는 어떻게 달라지는가?"**

## 독립변인

SAVANT는 세 가지 독립변인을 설정합니다. 각 변인은 독립적으로 제어되며, 조합하면 총 9가지 실험 조건이 만들어집니다.

| # | 독립변인 | 수준 | 설명 |
|---|----------|------|------|
| 1 | **처리 구조** | 순차 처리 (제어군) / 병렬 처리 (실험군) | 아두이노 나노 1대 vs 2×2 격자 4대 |
| 2 | **데이터 흐름 모드** | OS / WS | 실험군에만 적용. 제어군은 해당 없음 |
| 3 | **행렬 크기** | 2×2×2 / 2×2×4 / 2×4×2 | A행렬(S_R행 × T열) × B행렬(T행 × S_C열) 표기 |

<Note>
  행렬 크기 표기는 **S_R × T × S_C** 형식을 따릅니다. 예를 들어 2×2×4는 A가 2행 2열, B가 2행 4열인 행렬 곱을 의미하며, 결과 행렬 C는 2행 4열이 됩니다. S_R은 A의 행 수, T는 공통 차원(A의 열 = B의 행), S_C는 B의 열 수입니다.
</Note>

### 9가지 실험 조건

| 조건 번호 | 처리 구조 | 데이터 흐름 모드 | 행렬 크기 |
|-----------|-----------|-----------------|-----------|
| C-1 | 순차 (제어군) | — | 2×2×2 |
| C-2 | 순차 (제어군) | — | 2×2×4 |
| C-3 | 순차 (제어군) | — | 2×4×2 |
| E-OS-1 | 병렬 (실험군) | OS | 2×2×2 |
| E-OS-2 | 병렬 (실험군) | OS | 2×2×4 |
| E-OS-3 | 병렬 (실험군) | OS | 2×4×2 |
| E-WS-1 | 병렬 (실험군) | WS | 2×2×2 |
| E-WS-2 | 병렬 (실험군) | WS | 2×2×4 |
| E-WS-3 | 병렬 (실험군) | WS | 2×4×2 |

## 종속변인

종속변인은 각 실험 조건에서 측정하는 결과 지표입니다.

| # | 종속변인 | 측정 방법 | 단위 |
|---|----------|-----------|------|
| 1 | **처리 시간** | `micros()` 함수로 연산 시작~종료 시각 측정 | 마이크로초 (μs) |
| 2 | **이론적 클록 수 vs 실측 클록 수** | 공식(Nc)으로 이론값 계산, 코드 내 카운터로 실측 | 클록 횟수 |
| 3 | **메모리·통신 접근 횟수** | 배열 읽기/쓰기 횟수(제어군), UART 송수신 횟수(실험군) 코드 내 카운팅 | 횟수 |

<Note>
  실험군의 타이밍 측정은 D6(마스터 아두이노) 기준으로, PE(0,0)의 첫 번째 MAC 연산이 시작되기 **이전**에 타이머를 시작해야 공정한 비교가 됩니다. 타이머 시작 시점의 정확성은 실험 결과의 유효성에 직접 영향을 미칩니다.
</Note>

## 이론적 클록 수 공식

시스톨릭 어레이에서 모든 출력을 계산하는 데 필요한 이론적 클록 수는 다음 공식으로 예측합니다.

$$N_c = 2 \times S_R + S_C + T - 2$$

여기서 각 기호의 의미는 다음과 같습니다.

| 기호 | 의미 |
|------|------|
| S_R | 결과 행렬 C의 행 수 (= A 행렬의 행 수) |
| S_C | 결과 행렬 C의 열 수 (= B 행렬의 열 수) |
| T | 공통 차원 (A 열 수 = B 행 수) |

### 행렬 크기별 이론 클록 수 계산 예시

**2×2×2** (S_R=2, S_C=2, T=2):
$$N_c = 2 \times 2 + 2 + 2 - 2 = 6$$

**2×2×4** (S_R=2, S_C=4, T=2):
$$N_c = 2 \times 2 + 4 + 2 - 2 = 8$$

**2×4×2** (S_R=2, S_C=2, T=4):
$$N_c = 2 \times 2 + 2 + 4 - 2 = 8$$

<Tip>
  2×2×4와 2×4×2는 이론 클록 수가 동일(8)하지만, 데이터 흐름 방향이 다릅니다. 2×2×4는 열 방향으로 데이터가 더 많이 흐르고, 2×4×2는 공통 차원이 깊어 각 PE의 MAC 반복 횟수가 늘어납니다. 이 차이가 실측 시간에 어떻게 반영되는지가 흥미로운 분석 포인트입니다.
</Tip>

### OS와 WS 데이터 흐름에서의 공식 적용

- **OS(출력 고정) 모드**: 각 PE는 자신이 담당하는 C 행렬 원소의 누산값을 보유하고, A의 행 데이터가 좌→우로, B의 열 데이터가 위→아래로 흐릅니다. 위 공식은 마지막 PE까지 유효한 데이터가 도달하는 클록 수를 나타냅니다.
- **WS(가중치 고정) 모드**: 각 PE는 A 행렬의 원소(가중치)를 고정 보유하고, B 행렬의 열 벡터가 위→아래로 흐릅니다. WS 모드에서 C[0][p]는 PE(0,1)이 계산하며, 시리얼 모니터가 아닌 **I2C LCD**를 통해 결과를 검증합니다.

## 예상 결과와 반직관적 발견

직관적으로는 4대의 아두이노가 병렬로 처리하면 1대보다 빠를 것처럼 보입니다. 그러나 SAVANT에서는 **실험군이 제어군보다 느릴 수 있습니다.**

<Warning>
  아두이노 나노 간 통신은 SoftwareSerial(9600 baud)을 사용합니다. 1바이트 전송에 약 1밀리초가 소요되는 반면, SRAM 내 배열 접근은 나노초 단위입니다. 즉, 통신 오버헤드가 병렬 처리의 연산 이득을 압도할 가능성이 높습니다. 이는 실험 설계의 결함이 아니라, **소규모 저속 통신 환경에서 시스톨릭 어레이의 실제 한계**를 보여주는 중요한 발견입니다.
</Warning>

이 결과가 의미하는 바는 다음과 같습니다.

1. **시스톨릭 어레이의 장점은 고속 내부 통신(수 ns 단위)을 전제**합니다. TPU처럼 PE들이 동일 칩 위에 있어 통신 비용이 거의 없을 때 효율이 극대화됩니다.
2. **아두이노 수준의 UART 통신은 이 조건을 충족하지 못합니다**. 따라서 처리 시간 측면에서 실험군이 불리할 수 있습니다.
3. **그럼에도 메모리 접근 횟수와 클록 수는 구조적 차이를 보입니다.** 처리 시간이 더 길더라도, 이론 모델이 예측하는 클록 수 패턴은 측정 가능합니다.

결론적으로, 실험군이 느리다는 결과 자체가 "시스톨릭 어레이는 무용하다"는 의미가 아니라, **"올바른 통신 인프라 없이는 구조적 장점이 발현되지 않는다"**는 공학적 교훈을 담고 있습니다.

## 측정 계획

**총 측정 횟수: 최소 45회** (9가지 조건 × 5회 반복)

| 측정 단계 | 내용 |
|-----------|------|
| 각 조건당 반복 | 5회 측정 후 평균값과 표준편차 계산 |
| 이상값 처리 | 평균 ± 2σ를 벗어나는 값은 이상값으로 표시하고 별도 기록 (삭제하지 않음) |
| 결과 기록 형식 | 처리 시간(μs), 이론 클록 수, 실측 클록 수, 메모리/통신 접근 횟수를 표로 정리 |
| WS 모드 검증 | C[0][p] 값은 I2C LCD를 통해 PE(0,1)에서 직접 출력하여 정확성 확인 |

<Tip>
  5회 반복 측정에서 표준편차가 크게 나타나는 경우, 타이밍 스큐(각 아두이노의 클록 동기화 오차)가 원인일 수 있습니다. SAVANT는 타이밍 스큐를 별도로 보정하지 않으며, 이를 실험의 한계로 명시하고 결과 분석 시 고려합니다.
</Tip>

## 연구 정직성 원칙

SAVANT는 다음 원칙을 준수합니다.

1. **9가지 조건 전부 보고**: 결과가 가설에 불리하더라도 모든 조건의 측정값을 보고합니다. 유리한 조건만 선택 보고하는 **확증 편향**을 명시적으로 배제합니다.
2. **한계 명시**: 아두이노가 하버드 아키텍처인 점, 타이밍 스큐를 구현하지 않은 점, UART 통신이 결과에 미치는 영향 등을 결과 해석 시 명시합니다.
3. **과장 금지**: "AI 에너지 소비가 수십 배"와 같은 불정확한 수치 대신, 출처 있는 데이터(약 7배, de Vries 2023)를 사용합니다. "폰 노이만이 모든 문제의 근본 원인"이라는 논리 비약도 사용하지 않습니다.
4. **이상값 보존**: 측정 중 발생한 이상값은 삭제하지 않고 원인과 함께 기록합니다.

<Note>
  연구 정직성은 결과의 방향보다 중요합니다. 실험군이 제어군보다 느리게 측정되더라도, 그 원인을 엄밀하게 분석하고 구조적 함의를 도출하는 것이 이 탐구의 핵심 가치입니다.
</Note>
